基础命令

Git 基础命令是日常使用频率最高的操作。

工作区 (Working Directory) │ git add ▼ 暂存区 (Staging Area / Index) │ git commit ▼ 本地仓库 (Local Repository) │ git push ▼ 远程仓库 (Remote Repository)
Bash
# 配置
git config --global user.name "Your Name"
git config --global user.email "email@example.com"

# 初始化仓库
git init                            # 创建新仓库
git clone <url>                     # 克隆远程仓库

# 日常操作
git status                          # 查看状态
git add <file>                      # 添加文件到暂存区
git add .                           # 添加所有变更
git commit -m "feat: add login"     # 提交
git log --oneline                   # 查看简洁日志
git diff                            # 查看未暂存的变更
git diff --staged                   # 查看已暂存的变更

# 撤销操作
git restore <file>                  # 撤销工作区修改
git restore --staged <file>          # 撤销暂存
git reset --soft HEAD~1              # 撤销上次提交，保留修改
git reset --hard HEAD~1              # 撤销上次提交，丢弃修改

# 查看历史
git log --graph --oneline --all      # 图形化查看所有分支
git blame <file>                    # 查看每行最后的修改者

分支管理

分支是 Git 最强大的功能之一，让并行开发成为可能。

Bash
# 创建与切换分支
git branch feature-login              # 创建分支
git checkout feature-login            # 切换分支
git checkout -b feature-login         # 创建并切换（快捷）

# 合并分支
git checkout main                     # 先切回主分支
git merge feature-login               # 合并特性分支到 main

# 删除分支
git branch -d feature-login           # 删除已合并的分支
git branch -D feature-login           # 强制删除未合并的分支

# 变基 (rebase) - 保持线性历史
git checkout feature
git rebase main                       # 将 feature 基点到 main 顶端

# 解决冲突后继续 rebase
git rebase --continue                 # 解决冲突后继续
git rebase --abort                    # 放弃 rebase

# 储藏 (stash) - 临时保存工作
git stash                             # 暂存当前修改
git stash pop                         # 恢复最近一次暂存
git stash list                        # 查看所有暂存
git stash drop                        # 丢弃暂存

# 查看分支
git branch                            # 本地分支列表
git branch -a                         # 所有分支（含远程）
💡 merge vs rebase

merge：保留完整历史，产生合并提交，适合公共分支。

rebase：历史线性整洁，但会改写提交，适合个人特性分支。

⚠️ 不要对已推送的公共分支进行 rebase。

远程协作

Bash
# 远程仓库管理
git remote add origin <url>           # 添加远程仓库
git remote -v                         # 查看远程仓库
git remote remove origin              # 删除远程仓库

# 推送
git push origin main                  # 推送到远程 main
git push -u origin feature            # 推送到远端并建立追踪
git push --all                        # 推送所有分支
git push origin --delete feature      # 删除远程分支

# 拉取
git pull                              # 拉取并合并 (fetch + merge)
git pull --rebase                     # 拉取并变基 (推荐)
git fetch                             # 仅拉取，不自动合并

# 标签
git tag v1.0.0                        # 创建标签
git push origin --tags                # 推送所有标签
📘 推荐的拉取方式

使用 git pull --rebase 而非 git pull，可以避免产生多余的 merge commit，保持历史线性整洁。也可设置默认行为：

git config --global pull.rebase true

Git 工作流

🌿

Git Flow

经典工作流：main / develop / feature / release / hotfix 分支。适合有固定发布周期的项目

🚀

GitHub Flow

简化工作流：main + feature 分支 + PR。每个 feature 从 main 创建分支，通过 PR 合并回 main。持续部署友好

🔄

GitLab Flow

在 GitHub Flow 基础上增加环境分支（staging / production），支持环境部署

✅

主干开发 (Trunk-based)

短命特性分支 + 频繁合并到 main。适合 CI/CD 和 DevOps 文化

GitHub Flow 示意图: main ──●────●────────●────────────●── \ / / \ / / feature ──●──●──● (Pull Request → Merge)
