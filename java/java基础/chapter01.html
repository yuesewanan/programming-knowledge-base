功能单元 - 方法、注释

功能单元 - 方法、注释

在 Java 面向对象编程中，方法（Method） 和 注释（Comment） 是构成程序的基本功能单元。方法用于封装可重用的逻辑，注释用于提高代码的可读性和可维护性。

1. 方法（Method）

方法是执行特定任务的代码块，是类中实现功能的核心。

1.1 方法的基本语法

java
修饰符 返回类型 方法名(参数列表) {
    // 方法体
    return 返回值; // 可选，仅当返回类型非 void 时需要
}

1.2 方法的组成部分

组成部分说明
修饰符public、private、protected、static、final 等
返回类型void、int、String、自定义类型等
方法名遵循驼峰命名法，如 calculateTotal()
参数列表可选，多个参数用逗号分隔
方法体包含执行逻辑的代码块

1.3 方法示例

java
public class Calculator {
    // 无参无返回值方法
    public void sayHello() {
        System.out.println("Hello, World!");
    }
    // 带参有返回值方法
    public int add(int a, int b) {
        return a + b;
    }
    // 静态方法
    public static double multiply(double x, double y) {
        return x * y;
    }
    // 重载方法
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

1.4 方法的调用

java
Calculator calc = new Calculator();
calc.sayHello(); // 调用实例方法
int result = calc.add(3, 5); // 调用实例方法
double product = Calculator.multiply(2.5, 3.0); // 调用静态方法

1.5 方法重载（Overloading）

同一类中可以有多个同名方法，只要参数列表不同。

java
public class StringUtils {
    public String concat(String a, String b) {
        return a + b;
    }
    public String concat(String a, String b, String c) {
        return a + b + c;
    }
    public String concat(String a, int b) {
        return a + b;
    }
}

2. 注释（Comment）

注释用于解释代码逻辑、功能、参数、返回值等，是代码文档的重要组成部分。

2.1 单行注释

java
// 这是一个单行注释
int x = 10; // 变量 x 的初始值为 10

2.2 多行注释

java
/*
 * 这是一个多行注释
 * 可以跨越多行
 * 用于详细说明代码逻辑
 */
public void complexMethod() {
    // 方法体
}

2.3 文档注释（Javadoc）

用于生成 API 文档，支持 @param、@return、@throws 等标签。

java
/**
 * 计算两个整数的和
 * 
 * @param a 第一个整数
 * @param b 第二个整数
 * @return 两数之和
 */
public int add(int a, int b) {
    return a + b;
}

> 生成文档：使用 javadoc 命令生成 HTML 文档。

2.4 注释的最佳实践

原则说明
解释“为什么”，而不是“是什么”注释应解释设计意图，而非代码本身
保持简洁避免冗长，重点突出
及时更新代码修改后，注释应同步更新
避免注释无用代码删除无用代码，而非注释掉

3. 方法与注释的结合

良好的代码应具备清晰的方法命名和必要的注释。

java
/**
 * 根据用户输入计算折扣价格
 * 
 * @param originalPrice 原价
 * @param discountRate 折扣率（0~1）
 * @return 折后价格
 * @throws IllegalArgumentException 如果折扣率超出范围
 */
public double calculateDiscountedPrice(double originalPrice, double discountRate) {
    if (discountRate < 0 || discountRate > 1) {
        throw new IllegalArgumentException("折扣率必须在 0~1 之间");
    }
    return originalPrice * (1 - discountRate);
}

4. 总结

功能单元作用
方法封装可重用的逻辑，实现特定功能
注释提高代码可读性，便于维护和协作

> 方法是代码的“功能模块”，注释是代码的“说明书”

在实际开发中，应遵循“方法清晰、注释到位”的原则，编写可读、可维护、可复用的 Java 代码。

> 💡 建议： > - 方法名应具有语义，如 saveUser()、validateInput() > - 使用 Javadoc 注释公共方法 > - 避免过度注释，注释应服务于理解，而非替代代码

字面量

字面量

在 Java 中，字面量（Literal） 是指在代码中直接写出的、具有固定值的常量。字面量是程序中最基本的数据表示形式，用于初始化变量、传递参数、进行计算等。

1. 常见字面量类型

1.1 整数字面量

表示整数，支持十进制、八进制、十六进制。

类型示例说明
十进制123默认整数类型为 int
八进制0123以 0 开头，表示八进制
十六进制0x1A以 0x 或 0X 开头
长整型123L 或 123l后缀 L 表示 long 类型
java
int a = 123;        // 十进制 int
int b = 0123;       // 八进制 int (等于 83)
int c = 0x1A;       // 十六进制 int (等于 26)
long d = 123L;      // long 类型

1.2 浮点数字面量

表示浮点数，支持单精度（float）和双精度（double）。

类型示例说明
double3.14、1.0e-3默认为 double
float3.14f、1.0e-3f后缀 f 或 F 表示 float
java
double pi = 3.14159;
float speed = 10.5f;
double large = 1.23e10; // 科学计数法

1.3 字符字面量

表示单个字符，用单引号括起来。

java
char c1 = 'A';
char c2 = '1';
char c3 = '\n'; // 转义字符
char c4 = '\u0041'; // Unicode 字符

> 注意：char 是 16 位 Unicode 字符，不是字符串。

1.4 字符串字面量

表示字符串，用双引号括起来。

java
String s1 = "Hello";
String s2 = "World\n"; // 包含转义字符
String s3 = "中文"; // 支持 Unicode
String s4 = "Hello" + "World"; // 字符串拼接

> 字符串是对象，存储在字符串常量池中，具有不可变性。

1.5 布尔字面量

只有两个值：true 和 false。

java
boolean flag = true;
boolean isReady = false;

1.6 null 字面量

表示空引用，用于初始化引用类型变量。

java
String name = null;
Integer num = null;

2. 字面量的类型推断

Java 8+ 支持局部变量类型推断（var），字面量可用于推断类型。

java
var x = 123;        // 推断为 int
var y = 3.14;       // 推断为 double
var z = 'A';        // 推断为 char
var str = "Hello";  // 推断为 String
var flag = true;    // 推断为 boolean

> 注意：var 仅用于局部变量，不能用于类字段或方法参数。

3. 字面量的转义字符

在字符串和字符字面量中，可以使用转义字符。

转义字符含义
\'单引号
\"双引号
\\反斜杠
\n换行
\t制表符
\r回车
\b退格
\f换页
\uXXXXUnicode 字符（4 位十六进制）
java
String path = "C:\\Users\\User\\Desktop";
String message = "Hello\nWorld";
String quote = "He said: \"Hello!\"";

4. 字面量的注意事项

注意事项说明
整数默认为 int如果数值超出 int 范围，需加 L 后缀
浮点数默认为 double如果需要 float，必须加 f 后缀
字符串不可变字符串字面量存储在常量池，不可修改
null 不是对象null 表示空引用，不是任何类的实例

5. 总结

类型字面量示例说明
整数123, 0123, 0x1A, 123L支持多种进制
浮点数3.14, 3.14f, 1.0e-3默认为 double
字符'A', '\n', '\u0041'单引号，16 位 Unicode
字符串"Hello", "中文"双引号，不可变对象
布尔true, false逻辑值
nullnull空引用

> 字面量 = 程序中的“常量值”

字面量是 Java 编程中最基础的数据表示方式，理解其类型、格式和使用规则，是编写正确、高效代码的前提。

> 💡 建议： > - 使用 L 后缀明确长整型 > - 使用 f 后缀明确浮点型 > - 字符串拼接时注意性能（大量拼接建议用 StringBuilder） > - 避免在字符串中直接写特殊字符，使用转义字符

变量

变量

在 Java 中，变量（Variable） 是用于存储数据的内存空间的引用。每个变量都有一个类型、名称和值，用于在程序运行过程中保存和操作数据。

1. 变量的基本语法

java
数据类型 变量名 = 初始值;

1.1 基本数据类型变量

类型占用字节默认值说明
byte10-128 ~ 127

long80L超大整数
float40.0f单精度浮点数
double80.0双精度浮点数
char2'\u0000'Unicode 字符
boolean1falsetrue / false
java

short s = 1000;
int i = 100000;
long l = 10000000000L;
float f = 3.14f;
double d = 3.14159;

1.2 引用类型变量

引用类型变量存储的是对象的引用（内存地址），不是对象本身。

java
String name = "Alice";
int[] arr = {1, 2, 3};
List<String> list = new ArrayList<>();
Map<String, Integer> map = new HashMap<>();

2. 变量的分类

2.1 按作用域分类

类型说明示例
局部变量定义在方法、代码块或构造器中int x = 10;
实例变量定义在类中，方法外，没有 staticprivate String name;
静态变量使用 static 修饰，属于类public static int count = 0;

2.2 按生命周期分类

类型生命周期说明
局部变量方法执行期间方法结束即销毁
实例变量对象生命周期对象创建时初始化，对象销毁时释放

3. 变量的初始化

3.1 显式初始化

java
int age = 25;
String city = "Beijing";

3.2 隐式初始化（默认值）

java
int x; // 默认值为 0
boolean flag; // 默认值为 false
String s; // 默认值为 null

> 注意：局部变量必须显式初始化，否则编译报错。

java
// ❌ 错误：局部变量未初始化
int x;
System.out.println(x); // 编译错误

3.3 常量变量（final）

使用 final 修饰的变量为常量，一旦赋值不能修改。

java
final int MAX_SIZE = 100;
final String PI = "3.14159";

> 注意：final 变量必须在声明时或构造器中初始化。

4. 变量的作用域与生命周期

4.1 局部变量

java
public void test() {
    int a = 10; // 作用域：test 方法内
    {
        int b = 20; // 作用域：代码块内
        System.out.println(a + b); // 可访问
    }
    // System.out.println(b); // ❌ 错误：b 已超出作用域
}

4.2 实例变量

java
public class Person {
    private String name; // 实例变量，每个对象独立
    private int age;
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

4.3 静态变量

java
public class Counter {
    private static int count = 0; // 所有实例共享
    public Counter() {
        count++;
    }
    public static int getCount() {
        return count;
    }
}

5. 变量命名规范

类型命名规范示例
局部变量、实例变量驼峰命名法（lowerCamelCase）userName, totalAmount
静态变量、常量全大写，下划线分隔MAX_SIZE, PI_VALUE
类名首字母大写（UpperCamelCase）Person, OrderService
方法名驼峰命名法calculateTotal(), isValid()

6. 总结

特性说明
类型基本类型或引用类型
作用域局部、实例、静态
生命周期与作用域和对象生命周期相关
初始化局部变量必须显式初始化，其他有默认值
命名遵循驼峰命名、常量全大写等规范

> 变量 = 存储数据的“容器”

合理使用变量是编写清晰、高效 Java 程序的基础。应根据需求选择合适的作用域和类型，并遵循命名规范，提高代码可读性和可维护性。

> 💡 建议： > - 局部变量尽量靠近使用位置 > - 避免使用无意义的变量名（如 a, temp） > - 使用 final 修饰常量，提高代码安全性 > - 注意变量作用域，避免命名冲突

存储数据的原理 - 二进制 - 字符的存储 - ASCII 字符集

存储数据的原理 - 二进制 - 字符的存储 - ASCII 字符集

在计算机中，所有数据最终都以二进制形式存储。理解二进制和字符编码是掌握数据存储原理的基础。

1. 二进制基础

计算机使用二进制（Base-2）系统，由 0 和 1 组成。

1.1 位（bit）与字节（byte）

单位说明示例
位（bit）最小单位，0 或 11 位表示 2 种状态
字节（byte）8 位组成 1 字节1 字节 = 8 bit，可表示 256 种状态
text
01010101  // 一个字节，共 8 位

1.2 二进制与十进制转换

text
二进制: 1011 = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 8 + 0 + 2 + 1 = 11 (十进制)

1.3 数据存储单位

单位说明
KB1024 字节
MB1024 KB
GB1024 MB
TB1024 GB

2. 字符的存储 - ASCII 字符集

ASCII（American Standard Code for Information Interchange）是最早的字符编码标准，用于表示英文字符和控制字符。

2.1 ASCII 编码表（7 位）

ASCII 使用 7 位二进制表示字符，共 128 个字符（0~127）。

类型范围示例
控制字符0~31, 127回车、换行、制表符等
可打印字符32~126空格、字母、数字、符号

2.2 ASCII 在 Java 中的应用

在 Java 中，char 类型占用 2 字节（16 位），但可以存储 ASCII 字符（0~127）。

java
char c1 = 'A'; // ASCII 65
char c2 = '0'; // ASCII 48

> 注意：Java 使用 Unicode 编码，ASCII 是 Unicode 的子集。

3. 扩展字符集

由于 ASCII 仅支持英文，无法表示中文、日文等，因此发展出扩展编码：

> 数据存储 = 二进制 + 编码规则

八进制与十六进制

八进制与十六进制

在 Java 中，除了常用的十进制，还支持八进制和十六进制表示整数，常用于位运算、内存地址、颜色编码等场景。

1. 八进制（Octal）

八进制以 8 为基数，使用数字 0~7 表示。

1.1 表示方法

在 Java 中，八进制数以 0 开头。

java
int octal1 = 0123; // 八进制，等于十进制 83
int octal2 = 077;  // 八进制，等于十进制 63
int octal3 = 0;     // 八进制 0，等于十进制 0

1.2 八进制转十进制

text
0123 = 1×8² + 2×8¹ + 3×8⁰ = 64 + 16 + 3 = 83
077  = 7×8¹ + 7×8⁰ = 56 + 7 = 63

1.3 注意事项

  • 八进制不能包含 8 或 9
  • 0 开头的数字会被识别为八进制，即使只有一位（如 01 是八进制 1）
  • 建议避免使用八进制，除非有特殊需求（如权限设置）

2. 十六进制（Hexadecimal）

十六进制以 16 为基数，使用 0~9 和 A~F（或 a~f）表示。

2.1 表示方法

在 Java 中，十六进制数以 0x 或 0X 开头。

java
int hex1 = 0x1A;   // 十六进制，等于十进制 26
int hex2 = 0xFF;   // 十六进制，等于十进制 255
int hex3 = 0xABC;  // 十六进制，等于十进制 2748

2.2 十六进制转十进制

text
0x1A = 1×16¹ + 10×16⁰ = 16 + 10 = 26
0xFF = 15×16¹ + 15×16⁰ = 240 + 15 = 255
0xABC = 10×16² + 11×16¹ + 12×16⁰ = 2560 + 176 + 12 = 2748

2.3 常见应用场景

  • 颜色编码：#FF0000（红色） → 0xFF0000
  • 内存地址：0x7F800000（浮点数 NaN）
  • 位运算：0x0F 表示低 4 位掩码
java
// 颜色示例
int red = 0xFF0000; // 红色
int green = 0x00FF00; // 绿色
int blue = 0x0000FF; // 蓝色

3. 进制转换工具

Java 提供了内置方法进行进制转换：

java
// 十进制转十六进制
String hex = Integer.toHexString(255); // "ff"
// 十进制转八进制
String octal = Integer.toOctalString(63); // "77"
// 十六进制转十进制
int num = Integer.parseInt("1A", 16); // 26
// 八进制转十进制
int num2 = Integer.parseInt("77", 8); // 63

4. 总结

进制前缀数字范围应用场景
十进制无0~9日常计算
八进制00~7权限设置、旧系统
十六进制0x 或 0X0~9, A~F颜色、内存、位运算

> 八进制 = 0 开头，十六进制 = 0x 开头

在实际开发中，十六进制使用频率更高，尤其在图形、网络、系统编程中。八进制较少使用，但了解其语法有助于阅读旧代码或处理权限设置。

> 💡 建议： > - 使用 0x 表示十六进制，提高可读性 > - 避免在代码中直接使用八进制，除非必要 > - 使用 Integer.toHexString() 等方法进行进制转换，避免手动计算错误

数据类型

数据类型

在 Java 中，数据类型决定了变量可以存储的数据种类、占用内存大小和可执行的操作。Java 是强类型语言，所有变量必须声明类型。

基本数据类型是 Java 中最基础的数据类型，存储在栈中，直接存储值。

long80L超大整数需加 L 后缀

double80.0双精度浮点数默认浮点类型

2. 引用数据类型（Reference Types）

引用数据类型存储的是对象的引用（内存地址），对象本身存储在堆中。

2.1 常见引用类型

类型说明示例
类（Class）自定义类Person p = new Person();

3. 数据类型转换

3.1 自动类型转换（隐式）

从低精度到高精度，自动转换。

java
int a = 10;

double c = a; // 自动转换

3.2 强制类型转换（显式）

从高精度到低精度，需显式转换，可能丢失精度。

java
double d = 3.14;
int i = (int) d; // 强制转换，结果为 3

3.3 包装类转换（装箱/拆箱）

基本类型 ↔ 包装类（自动装箱/拆箱）

java
Integer i = 10; // 自动装箱
int x = i;      // 自动拆箱

标识符与关键字

标识符与关键字

在 Java 中，标识符（Identifier） 是用于命名类、方法、变量、参数等的名称，而关键字（Keyword） 是 Java 语言预定义的保留字，具有特殊含义，不能用作标识符。

1. 标识符（Identifier）

1.1 标识符的命名规则

规则说明
首字符必须是字母、下划线 _ 或美元符号 $
后续字符可以是字母、数字、下划线 _ 或美元符号 $
大小写敏感myVar 和 myvar 是不同的标识符
不能是关键字如 class、public、int 等不能用作标识符

1.2 命名规范（推荐）

类型命名规范示例
类名首字母大写（UpperCamelCase）Person, OrderService
方法名、变量名首字母小写（lowerCamelCase）calculateTotal(), userName
常量名全大写，下划线分隔MAX_SIZE, PI_VALUE
包名全小写，多级用点分隔com.example.util

1.3 合法与非法标识符示例

java
// ✅ 合法标识符
int age;
String userName;
double _price;
int $count;
// ❌ 非法标识符
int 123age; // 以数字开头
int class;  // 关键字
int my-var; // 包含非法字符 -

3. 标识符与关键字的使用建议

  • 避免使用关键字作为变量名或类名
