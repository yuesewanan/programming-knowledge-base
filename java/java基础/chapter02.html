方法详解

方法详解

在 Java 中，方法（Method） 是类中用于封装特定功能的代码块，是实现程序逻辑的核心单元。方法支持参数传递、返回值、重载、访问控制等特性。

1. 方法的基本语法

java
修饰符 返回类型 方法名(参数列表) {
    // 方法体
    return 返回值; // 可选，仅当返回类型非 void 时需要
}

1.1 组成部分

组成部分说明
修饰符public、private、protected、static、final 等
返回类型void、int、String、自定义类型等
方法名遵循驼峰命名法，如 calculateTotal()
参数列表可选，多个参数用逗号分隔
方法体包含执行逻辑的代码块

2. 方法的分类

2.1 按返回类型分类

类型说明示例
无返回值返回类型为 voidvoid printHello()
有返回值返回具体类型int add(int a, int b)

2.2 按访问修饰符分类

修饰符说明
public公有，任何地方可访问
private私有，仅本类可访问
protected受保护，子类和同包类可访问
默认（包私有）同包类可访问

2.3 按是否静态分类

类型说明
实例方法需要对象调用obj.method()
静态方法通过类名调用ClassName.method()

3. 方法的参数

3.1 基本类型参数

java
public int add(int a, int b) {
    return a + b;
}

3.2 引用类型参数

java
public void printUser(User user) {
    System.out.println(user.getName());
}

3.3 可变参数（Varargs）

从 Java 5 开始支持可变参数，用 ... 表示。

java
public int sum(int... numbers) {
    int total = 0;
    for (int num : numbers) {
        total += num;
    }
    return total;
}
// 调用示例
sum(1, 2, 3); // 6
sum(); // 0

4. 方法的返回值

4.1 返回基本类型

java
public int getMax(int a, int b) {
    return a > b ? a : b;
}

4.2 返回引用类型

java
public User getUserById(int id) {
    // 查询数据库
    return new User("Alice", 25);

4.3 返回 void

java
public void displayMessage(String msg) {
    System.out.println(msg);
}

5. 方法重载（Overloading）

同一类中可以有多个同名方法，只要参数列表不同（类型、数量、顺序不同）。

java
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    public double add(double a, double b) {
        return a + b;
    }
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}

> 调用时根据参数自动匹配

6. 方法的调用

6.1 实例方法调用

java
Calculator calc = new Calculator();
int result = calc.add(3, 5);

6.2 静态方法调用

java
int result = Calculator.add(3, 5); // 通过类名调用

6.3 方法链（Method Chaining）

通过返回 this 实现方法链。

java
public class StringBuilderExample {
    private StringBuilder sb;
    public StringBuilderExample() {
        sb = new StringBuilder();
    }
    public StringBuilderExample append(String str) {
        sb.append(str);
        return this;
    }
    public StringBuilderExample append(int num) {
        sb.append(num);
        return this;
    }
    public String toString() {
        return sb.toString();
    }
    // 使用示例
    public static void main(String[] args) {
        StringBuilderExample builder = new StringBuilderExample();
        builder.append("Hello").append(" ").append("World").append(123);
        System.out.println(builder.toString()); // 输出: Hello World123
    }
}

静态方法不能访问实例变量除非通过对象引用

特性说明
封装功能方法封装可重用的逻辑

方法的注意事项

方法的注意事项

在 Java 开发中，方法是实现业务逻辑的核心单元。合理设计和使用方法可以显著提高代码的可读性、可维护性和可复用性。以下是编写和调用方法时需要注意的关键事项。

1. 方法命名规范

  • 使用动词开头：如 calculateTotal()、validateInput()、saveUser()
  • 遵循驼峰命名法：getUserName()、setAge()
  • 避免无意义名称：如 doSomething()、process()，应具体化
  • 避免与类名或关键字冲突
> ✅ 推荐：createOrder() > ❌ 不推荐：func1()、method()、run()（除非是主方法）

2. 方法参数设计

2.1 参数数量

  • 尽量少：参数越多，方法越难理解
  • 建议不超过 3 个参数，超过时考虑使用对象封装
java
// ❌ 参数过多
public void sendEmail(String to, String subject, String body, boolean isHtml, String cc, String bcc) { ... }
// ✅ 推荐：封装为对象
public void sendEmail(EmailConfig config) { ... }

2.2 参数类型

  • 避免使用 Object 类型，除非必要
  • 优先使用具体类型，提高类型安全

3. 返回值处理

  • 明确返回值含义：返回 null 应有文档说明
  • 避免返回 null，可使用 Optional（Java 8+）或默认值
  • 返回值类型应与方法功能匹配
java
// ❌ 不推荐：返回 null 表示失败
public User getUserById(int id) {
    return null; // 无用户时返回 null
}
// ✅ 推荐：使用 Optional
public Optional<User> getUserById(int id) {
    return Optional.ofNullable(userDao.findById(id));
}

4. 方法重载注意事项

  • 参数列表必须不同：类型、数量、顺序不同
  • 避免重载歧义：如 add(int a, int b) 和 add(Integer a, Integer b) 会冲突（自动装箱）
java
// ❌ 重载冲突（自动装箱导致）
public void add(int a, int b) { ... }
public void add(Integer a, Integer b) { ... } // 编译错误
// ✅ 正确重载
public void add(int a, int b) { ... }
public void add(double a, double b) { ... }

5. 方法访问控制

  • 私有方法：仅在类内使用，避免暴露内部实现
  • 公有方法：对外提供接口，应有清晰文档
  • 静态方法：用于工具类，不依赖实例状态
> ✅ 推荐：工具类方法用 static，如 StringUtils.isEmailValid()

6. 方法体设计

  • 单一职责原则：一个方法只做一件事
  • 避免过长方法：建议不超过 20 行
  • 减少嵌套层级：避免多层 if-else 或 try-catch
java
// ❌ 过长方法
public void processOrder(Order order) {
    if (order != null && order.getItems() != null) {
        for (Item item : order.getItems()) {
            if (item.getPrice() > 0) {
                // 处理逻辑...
            }
        }
    }
    // 更多逻辑...
}
// ✅ 拆分方法
public void processOrder(Order order) {
    if (order == null) return;
    validateOrder(order);
    calculateTotal(order);
    saveOrder(order);
}
private void validateOrder(Order order) { ... }
private void calculateTotal(Order order) { ... }
private void saveOrder(Order order) { ... }

7. 异常处理

  • 明确抛出异常：如 IllegalArgumentException、NullPointerException
  • 避免捕获所有异常：catch (Exception e) 应谨慎使用
  • 使用 throws 声明异常，或在方法内处理
java
public void divide(int a, int b) throws ArithmeticException {
    if (b == 0) {
        throw new ArithmeticException("除数不能为零");
    }
    return a / b;
}

8. 性能与资源管理

  • 避免在方法中创建大量对象，尤其在循环中
  • 及时关闭资源：如文件、数据库连接（使用 try-with-resources）
  • 避免重复计算：缓存结果或使用静态变量

9. 文档与注释

  • 使用 Javadoc 注释公共方法
  • 说明参数、返回值、异常、使用场景
java
/**
 * 计算两个整数的和
 * 
 * @param a 第一个整数
 * @param b 第二个整数
 * @return 两数之和
 * @throws IllegalArgumentException 如果参数为负数
 */
public int add(int a, int b) {
    if (a < 0 || b < 0) {
        throw new IllegalArgumentException("参数不能为负数");
    }
    return a + b;
}

10. 总结

注意事项说明
命名规范动词开头，驼峰命名
参数设计尽量少，避免 Object
返回值避免 null，使用 Optional
重载参数列表必须不同
访问控制合理使用 public/private/static
方法体单一职责，避免过长
异常处理明确抛出，避免捕获所有异常
性能避免资源浪费，及时释放
注释使用 Javadoc 说明方法

> 方法 = 程序的“功能模块”，设计好方法 = 设计好程序

合理设计方法，是编写高质量、可维护 Java 代码的基础。

> 💡 建议： > - 使用 IDE 的代码分析工具检查方法长度、参数数量 > - 遵循团队编码规范 > - 定期重构过长或复杂的方法

自动与强制类型转换

自动与强制类型转换

在 Java 中，类型转换是将一个数据类型转换为另一个数据类型的过程。根据转换方式，分为自动类型转换（隐式） 和 强制类型转换（显式）。

1. 自动类型转换（隐式转换）

当从低精度类型向高精度类型转换时，Java 自动进行类型转换，无需显式声明。

1.1 基本数据类型自动转换

低精度 → 高精度示例
byte → short → int → long → float → doubleint a = 10; long b = a;
char → int → long → float → doublechar c = 'A'; int i = c;
java
byte b = 10;
short s = b; // 自动转换
int i = s;   // 自动转换
long l = i;  // 自动转换
double d = l; // 自动转换

> 注意：char 转 int 是自动的，因为 char 是 16 位，int 是 32 位。

2. 强制类型转换（显式转换）

当从高精度类型向低精度类型转换时，必须显式使用类型转换操作符 ()，否则编译报错。

2.1 基本数据类型强制转换

java
double d = 3.14;
int i = (int) d; // 强制转换，结果为 3（截断，非四舍五入）
long l = 10000000000L;
int i = (int) l; // 可能溢出，结果为负数
float f = 3.14f;
byte b = (byte) f; // 可能溢出或精度丢失

> 注意：强制转换可能导致数据丢失或溢出，需谨慎使用。

3. 包装类与基本类型的转换（装箱/拆箱）

Java 5+ 支持自动装箱（Boxing）和拆箱（Unboxing）。

3.1 自动装箱

java
Integer i = 10; // 自动装箱：int → Integer
Double d = 3.14; // 自动装箱：double → Double

3.2 自动拆箱

java
int x = i; // 自动拆箱：Integer → int
double y = d; // 自动拆箱：Double → double

> 注意：null 拆箱会抛出 NullPointerException。

java
Integer i = null;
int x = i; // ❌ NullPointerException

4. 引用类型转换

4.1 向上转型（Upcasting）

从子类对象转换为父类引用，自动完成。

java
Animal animal = new Dog(); // 自动向上转型

4.2 向下转型（Downcasting）

从父类引用转换为子类对象，需显式转换，可能抛出 ClassCastException。

java
Animal animal = new Dog();
Dog dog = (Dog) animal; // 显式向下转型

> 建议：使用 instanceof 判断类型安全。

java
if (animal instanceof Dog) {
    Dog dog = (Dog) animal;
}

5. 类型转换注意事项

注意事项说明
自动转换安全低精度 → 高精度，无数据丢失
强制转换风险高精度 → 低精度，可能溢出或精度丢失
装箱拆箱性能频繁装箱拆箱影响性能，建议避免
null 拆箱危险null 拆箱会抛出异常
向下转型需判断使用 instanceof 避免 ClassCastException

6. 总结

转换类型说明示例
自动转换低精度 → 高精度int → long
强制转换高精度 → 低精度(int) double
装箱基本类型 → 包装类int → Integer
拆箱包装类 → 基本类型Integer → int
向上转型子类 → 父类Dog → Animal
向下转型父类 → 子类(Dog) Animal

> 类型转换 = 数据的“格式转换”，需谨慎处理精度和安全

> 💡 建议： > - 优先使用自动转换，避免手动强制转换 > - 避免频繁装箱拆箱，尤其在循环中 > - 向下转型前使用 instanceof 判断 > - 使用 Optional 或 null 检查避免拆箱异常

表达式的自动类型提升

表达式的自动类型提升

在 Java 中，当表达式中包含不同数据类型的值时，Java 会自动进行类型提升（Type Promotion），将较低精度的类型提升为较高精度的类型，以保证计算的准确性。

1. 类型提升规则

Java 在表达式计算时遵循以下提升规则：

1.1 基本类型提升顺序

text
byte → short → int → long → float → double

> 注意：char 也会被提升为 int。

1.2 提升规则详解

表达式类型提升结果
byte + int提升为 int
short + long提升为 long
int + float提升为 float
long + double提升为 double
char + int提升为 int
byte + byte提升为 int（两个 byte 相加，结果为 int）

2. 表达式提升示例

java
byte b1 = 10;
byte b2 = 20;
int result = b1 + b2; // ❌ 不能直接赋值给 byte，因为表达式提升为 int
// ✅ 正确写法：显式转换
byte result = (byte) (b1 + b2); // 强制转换回 byte
// 示例 2：char 与 int 相加
char c = 'A'; // ASCII 65
int i = 10;
int sum = c + i; // 提升为 int，结果为 75
// 示例 3：混合类型
double d = 3.14;
int i = 10;
double result = d + i; // i 提升为 double，结果为 13.14

3. 为什么需要类型提升？

  • 避免数据丢失：如 byte + byte 若不提升，可能溢出
  • 保证计算精度：如 int + float 提升为 float，避免截断
  • 统一计算环境：所有操作数在相同类型下进行计算

4. 常见陷阱

4.1 byte + byte 不能直接赋值给 byte

java
byte a = 10;
byte b = 20;
byte c = a + b; // ❌ 编译错误！表达式提升为 int，不能赋值给 byte

✅ 正确写法：

java
byte c = (byte) (a + b); // 显式转换

4.2 char 与 int 混合运算

java
char c = 'A'; // 65
int i = 10;
int result = c + i; // 75，char 提升为 int

5. 总结

表达式提升结果说明
byte + byteint两个 byte 相加，结果为 int
char + intintchar 提升为 int
int + longlongint 提升为 long
float + doubledoublefloat 提升为 double

> 表达式自动类型提升 = 保证计算精度和安全

> 💡 建议： > - 理解表达式提升规则，避免编译错误 > - 在混合类型运算后，如需赋值给低精度类型，使用显式转换 > - 避免在循环中频繁进行类型转换，影响性能

输入-输出

输入-输出

在 Java 中，输入（Input）和输出（Output）是程序与外部环境交互的基础。Java 提供了丰富的 I/O 类库，支持控制台、文件、网络等多种输入输出方式。

1. 控制台输入

1.1 使用 Scanner 类（推荐）

java
import java.util.Scanner;
public class InputExample {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("请输入姓名: ");
        String name = scanner.nextLine();
        System.out.print("请输入年龄: ");
        int age = scanner.nextInt();
        System.out.println("姓名: " + name + ", 年龄: " + age);
        scanner.close(); // 关闭 Scanner，防止资源泄漏
    }
}

> 注意：Scanner 使用后必须调用 close()，或使用 try-with-resources。

java
try (Scanner scanner = new Scanner(System.in)) {
    // 使用 scanner
}

1.2 使用 BufferedReader（更高效）

java
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
public class InputExample {
    public static void main(String[] args) throws IOException {
        BufferedReader reader = new BufferedReader(new InputStreamReader(System.in));
        System.out.print("请输入姓名: ");
        String name = reader.readLine();
        System.out.print("请输入年龄: ");
        int age = Integer.parseInt(reader.readLine());
        System.out.println("姓名: " + name + ", 年龄: " + age);
        reader.close();
    }
}

2. 控制台输出

2.1 使用 System.out.println() 和 System.out.print()

java
System.out.println("Hello, World!"); // 换行输出
System.out.print("Hello, "); // 不换行输出
System.out.println("World!");

2.2 格式化输出（printf）

java
int age = 25;
double price = 3.14;
System.out.printf("年龄: %d, 价格: %.2f%n", age, price);
// 输出: 年龄: 25, 价格: 3.14

> 格式说明符：

  • %d：整数
  • %f：浮点数
  • %.2f：保留两位小数
  • %s：字符串
  • %n：换行（平台无关）

3. 文件输入输出

3.1 写入文件（FileWriter）

java
import java.io.FileWriter;
import java.io.IOException;
public class FileWriteExample {
    public static void main(String[] args) {
        try (FileWriter writer = new FileWriter("output.txt")) {
            writer.write("Hello, File!\n");
            writer.write("This is a test.\n");
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

3.2 读取文件（FileReader）

java
import java.io.FileReader;
import java.io.IOException;
public class FileReadExample {
    public static void main(String[] args) {
        try (FileReader reader = new FileReader("input.txt")) {
            int character;
            while ((character = reader.read()) != -1) {
                System.out.print((char) character);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

4. 使用 try-with-resources（推荐）

自动关闭资源，避免资源泄漏。

java
try (Scanner scanner = new Scanner(System.in);
     FileWriter writer = new FileWriter("output.txt")) {
    String input = scanner.nextLine();
    writer.write(input);
} catch (IOException e) {
    e.printStackTrace();
}

基本运算符 - 自增自减运算符

基本运算符 - 自增自减运算符

在 Java 中，自增（++） 和 自减（--） 运算符用于将变量的值增加或减少 1。它们分为前缀和后缀两种形式，行为略有不同。

1. 自增运算符 ++

1.1 前缀自增 ++i

先将变量加 1，再返回新值。

java
int i = 5;
int result = ++i; // i 先变为 6，然后 result = 6
System.out.println(i); // 输出: 6
System.out.println(result); // 输出: 6

1.2 后缀自增 i++

先返回原值，再将变量加 1。

java
int i = 5;
int result = i++; // result = 5，然后 i 变为 6
System.out.println(i); // 输出: 6
System.out.println(result); // 输出: 5

2. 自减运算符 --

2.1 前缀自减 --i

先将变量减 1，再返回新值。

java
int i = 5;
int result = --i; // i 先变为 4，然后 result = 4
System.out.println(i); // 输出: 4
System.out.println(result); // 输出: 4

2.2 后缀自减 i--

先返回原值，再将变量减 1。

java
int i = 5;
int result = i--; // result = 5，然后 i 变为 4
System.out.println(i); // 输出: 4
System.out.println(result); // 输出: 5

3. 自增自减运算符的注意事项

3.1 不能用于常量或表达式

java
// ❌ 错误：不能对常量使用自增
int x = 5 + ++1; // 编译错误
// ❌ 错误：不能对表达式使用自增
int a = 5, b = 10;
int c = a++ + b++; // ❌ 语法错误（虽然 a++ 和 b++ 本身合法，但不能用于表达式中？实际是合法的，但需注意顺序）

> 注意：a++ + b++ 是合法的，但执行顺序由左到右，且 a++ 返回原值，b++ 返回原值。

java
int a = 5, b = 10;
int c = a++ + b++; // c = 5 + 10 = 15，然后 a=6, b=11

3.2 在赋值语句中的使用

java
int i = 5;
i = i++; // i = 5，然后 i 变为 6 → 最终 i = 6？ ❌ 实际是 i = 5，因为 i++ 返回原值
System.out.println(i); // 输出: 6
// 但注意：
int j = 5;
j = ++j; // j = 6，然后 j = 6 → 最终 j = 6

> 关键点：i = i++ 的结果是 i 的值不变（因为赋值的是 i++ 返回的原值），但 i 本身会自增。

4. 自增自减的性能与建议

  • 性能：自增自减是原子操作，但多线程环境下需加锁或使用 AtomicInteger
  • 建议：
- 单独使用时，优先使用 i++ 或 ++i，语义清晰 - 在赋值语句中，避免 i = i++，容易引起混淆 - 在循环中，使用 i++ 更常见（如 for (int i = 0; i < 10; i++)）

5. 总结

运算符说明示例结果

i++后缀自增int x = i++;先赋值，再加 1
--i前缀自减int x = --i;先减 1，再赋值
i--后缀自减int x = i--;先赋值，再减 1

> 自增自减 = 快速加减 1 的便捷运算符

> 💡 建议： > - 在循环中使用 i++ 或 ++i，语义清晰 > - 避免在复杂表达式中使用自增自减，易出错 > - 多线程环境下，使用 AtomicInteger 替代普通 int 自增

赋值运算符-三元运算符

赋值运算符-三元运算符

在 Java 中，赋值运算符用于将值赋给变量，而三元运算符（也称条件运算符）是 Java 中唯一一个三目运算符，用于根据条件表达式的真假返回两个值中的一个。

1. 赋值运算符

1.1 基本赋值运算符 =

将右边的值赋给左边的变量。

java
int a = 10; // 将 10 赋值给变量 a

1.2 复合赋值运算符

复合赋值运算符将赋值与算术运算结合，简化代码。

运算符作用示例等价写法
+=加法赋值a += 5;a = a + 5;
-=减法赋值a -= 3;a = a - 3;
*=乘法赋值a *= 2;a = a * 2;
/=除法赋值a /= 4;a = a / 4;
%=取模赋值a %= 3;a = a % 3;

> 注意：复合赋值运算符会自动进行类型转换，避免溢出问题。

java
byte b = 10;
b += 5; // 等价于 b = (byte)(b + 5); ✅ 自动类型转换
// b = b + 5; ❌ 编译错误，因为 b + 5 是 int 类型

2. 三元运算符

2.1 语法

java
boolean_expression ? value_if_true : value_if_false
  • 如果 boolean_expression 为 true，返回 value_if_true
  • 如果为 false，返回 value_if_false

2.2 示例

java
int a = 10;
int b = 20;
int max = a > b ? a : b; // 如果 a > b 成立，则 max = a，否则 max = b

在这个例子中，max 的值为 20。

2.3 嵌套使用

三元运算符可以嵌套使用，实现多条件判断。

java
int score = 85;
String grade = score >= 90 ? "A" :
               score >= 80 ? "B" :
               score >= 70 ? "C" :
               score >= 60 ? "D" : "F";

输出：grade = "B"

3. 使用注意事项

3.1 不能用于常量或表达式

java
// ❌ 错误：不能对常量使用赋值运算符
int x = 5 + 10 = 20; // 编译错误

3.2 三元运算符的类型兼容性

三元运算符的两个分支必须能兼容，否则会报错。

java
int a = 10;
String result = a > 5 ? "大" : 100; // ❌ 编译错误，String 和 int 不兼容

> 正确写法：

java
String result = a > 5 ? "大" : "小"; // ✅ 两个分支都是 String

+=加法赋值a += 5;
? :三元运算符a > b ? a : b

> 赋值运算符 = 快速赋值的便捷方式 > 三元运算符 = 简洁的条件表达式

> 💡 建议： > - 使用复合赋值运算符简化代码 > - 三元运算符适合简单条件判断，避免嵌套过深 > - 注意类型兼容性，避免编译错误

逻辑运算符

逻辑运算符

在 Java 中，逻辑运算符用于连接多个布尔表达式，并根据它们的真假值返回一个布尔结果。逻辑运算符是条件判断和流程控制的重要组成部分。

1. 逻辑与运算符 &&

&& 是逻辑与运算符，只有当两个操作数都为 true 时，结果才为 true。

java
boolean a = true;
boolean b = false;
boolean result = a && b; // result = false

短路特性

&& 具有短路求值特性：如果第一个操作数为 false，则不会计算第二个操作数。

java
int x = 0;
boolean result = (x > 0) && (10 / x > 2); // 不会执行除法，避免除零异常

> 注意：&& 可以防止潜在的运行时错误，如空指针或除零。

2. 逻辑或运算符 ||

|| 是逻辑或运算符，只要有一个操作数为 true，结果就为 true。

java
boolean a = true;
boolean b = false;
boolean result = a || b; // result = true

短路特性

|| 同样具有短路求值特性：如果第一个操作数为 true，则不会计算第二个操作数。

java
int x = 0;
boolean result = (x > 0) || (10 / x > 2); // 不会执行除法，避免除零异常

3. 逻辑非运算符 !

! 是逻辑非运算符，用于取反布尔值。

java
boolean a = true;
boolean result = !a; // result = false

4. 逻辑运算符的优先级

逻辑运算符的优先级从高到低为：

1. !（逻辑非） 2. &&（逻辑与） 3. ||（逻辑或）

java
boolean a = true;
boolean b = false;
boolean c = true;
// 等价于: (a && b) || c
boolean result = a && b || c; // result = true

> 建议：使用括号明确优先级，提高代码可读性。

5. 逻辑运算符与位运算符的区别

运算符类型是否短路说明
&按位与 / 逻辑与否总是计算两个操作数
&&逻辑与是短路求值
按位或 / 逻辑或否总是计算两个操作数
逻辑或是短路求值

> 推荐使用 && 和 ||，因为它们更安全、高效。

6. 实际应用示例

示例 1：用户登录验证

java
boolean isUsernameValid = true;
boolean isPasswordValid = false;
if (isUsernameValid && isPasswordValid) {
    System.out.println("登录成功");
} else {
    System.out.println("登录失败");
}

示例 2：空值检查

java
String name = null;
if (name != null && name.length() > 0) {
    System.out.println("姓名有效");
} else {
    System.out.println("姓名无效");
}

> 使用 && 可以避免 NullPointerException。

逻辑或短路求值

综合实战 - 身体健康指数计算

综合实战 - 身体健康指数计算

在实际开发中，我们经常需要将多个知识点综合运用，解决一个具体问题。本节通过一个综合实战项目：计算身体健康指数（BMI），来巩固前面所学的输入输出、运算符、数据类型、条件判断等知识。

1. 项目目标

根据用户的身高和体重，计算其 身体质量指数（BMI），并给出健康建议。

1.1 BMI 计算公式

$$ \text{BMI} = \frac{\text{体重（kg）}}{\text{身高（m）}^2} $$

1.2 BMI 分类标准

BMI 范围健康状态
< 18.5体重过轻
18.5 - 23.9正常范围
24.0 - 27.9超重
≥ 28.0肥胖

2. 实现步骤

2.1 输入用户信息

使用 Scanner 获取用户的身高（单位：米）和体重（单位：千克）。

java
import java.util.Scanner;
public class BMICalculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("请输入身高（米）: ");
        double height = scanner.nextDouble();
        System.out.print("请输入体重（千克）: ");
        double weight = scanner.nextDouble();
    }
}

2.2 计算 BMI

java
double bmi = weight / (height * height);

2.3 输出结果并给出建议

使用 System.out.printf 输出格式化结果，并通过 if-else 判断健康状态。

java
System.out.printf("您的 BMI 为: %.2f%n", bmi);
if (bmi < 18.5) {
    System.out.println("建议：体重过轻，注意营养摄入。");
} else if (bmi >= 18.5 && bmi < 24.0) {
    System.out.println("建议：体重正常，保持健康生活方式。");
} else if (bmi >= 24.0 && bmi < 28.0) {
    System.out.println("建议：超重，建议控制饮食和增加运动。");
} else {
    System.out.println("建议：肥胖，建议咨询医生或营养师。");
}

3. 完整代码

java
import java.util.Scanner;
public class BMICalculator {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("请输入身高（米）: ");
        double height = scanner.nextDouble();
        System.out.print("请输入体重（千克）: ");
        double weight = scanner.nextDouble();
        double bmi = weight / (height * height);
        System.out.printf("您的 BMI 为: %.2f%n", bmi);
        if (bmi < 18.5) {
            System.out.println("建议：体重过轻，注意营养摄入。");
        } else if (bmi >= 18.5 && bmi < 24.0) {
            System.out.println("建议：体重正常，保持健康生活方式。");
        } else if (bmi >= 24.0 && bmi < 28.0) {
            System.out.println("建议：超重，建议控制饮食和增加运动。");
        } else {
            System.out.println("建议：肥胖，建议咨询医生或营养师。");
        }
        scanner.close();
    }
}
