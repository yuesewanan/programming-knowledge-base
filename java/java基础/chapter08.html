抽象类 - 认识抽象类

抽象类 - 认识抽象类

在 Java 中，抽象类（Abstract Class） 是一种特殊的类，它不能被实例化，只能被继承。抽象类通常用于定义一组公共的属性和方法，供子类继承和实现。抽象类是面向对象编程的重要特性之一，它提高了代码的复用性和可扩展性。

1. 抽象类的基本语法

1.1 定义抽象类

java
abstract class Animal {
    // 抽象方法
    abstract void makeSound();
    // 普通方法
    void sleep() {
        System.out.println("动物在睡觉");
    }
}

> 说明：

  • abstract 关键字用于定义抽象类
  • 抽象类可以包含抽象方法和普通方法
  • 抽象方法没有方法体，必须在子类中实现

1.2 抽象方法

java
abstract class Animal {
    // 抽象方法
    abstract void makeSound();
    // 普通方法
    void sleep() {
        System.out.println("动物在睡觉");
    }
}

> 说明：

  • abstract 关键字用于定义抽象方法
  • 抽象方法没有方法体，必须在子类中实现

2. 抽象类的特点

2.1 不能被实例化

抽象类不能被实例化，只能被继承。

java
abstract class Animal {
    abstract void makeSound();
}
public class Main {
    public static void main(String[] args) {
        // Animal animal = new Animal(); // ❌ 编译错误：不能实例化抽象类
    }
}

> 说明：

  • Animal 是抽象类，不能被实例化
  • 必须通过子类实例化

2.2 可以包含普通方法

抽象类可以包含普通方法，这些方法可以被子类继承和使用。

java
abstract class Animal {
    // 抽象方法
    abstract void makeSound();
    // 普通方法
    void sleep() {
        System.out.println("动物在睡觉");
    }
}

> 说明：

  • sleep() 是普通方法，可以被子类继承和使用

2.3 可以包含抽象方法

抽象类可以包含抽象方法，这些方法必须在子类中实现。

java
abstract class Animal {
    // 抽象方法
    abstract void makeSound();
    // 普通方法
    void sleep() {
        System.out.println("动物在睡觉");
    }
}

> 说明：

  • makeSound() 是抽象方法，必须在子类中实现

3. 抽象类的应用场景

3.1 定义公共行为

抽象类可以定义一组公共的行为，供子类继承和实现。

java
abstract class Animal {
    abstract void makeSound();
    void sleep() {
        System.out.println("动物在睡觉");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("喵喵");
    }
}

> 说明：

  • Animal 定义了 makeSound() 和 sleep() 方法
  • Dog 和 Cat 实现了 makeSound() 方法

3.2 定义公共属性

抽象类可以定义一组公共的属性，供子类继承和使用。

java
abstract class Animal {
    protected String name;
    protected int age;
    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }
    abstract void makeSound();

        System.out.println("动物在睡觉");
    }
}

> 说明：

  • name 和 age 是公共属性，可以被子类继承和使用

4.1 抽象类不能被实例化

抽象类不能被实例化，只能被继承。

java

}

  • Animal 是抽象类，不能被实例化

4.2 抽象方法必须在子类中实现

抽象方法必须在子类中实现，否则会编译错误。

java

    abstract void makeSound();

    void makeSound() {

    }

  • makeSound() 是抽象方法，必须在子类中实现

5. 总结

java

抽象类 - 模板方法设计模式

抽象类 - 模板方法设计模式

在 Java 中，模板方法设计模式（Template Method Pattern） 是一种行为型设计模式，它定义了一个算法的骨架，将一些步骤延迟到子类中实现。模板方法模式通过抽象类来定义算法的框架，子类通过继承抽象类并实现特定步骤来完成算法。

1. 模板方法设计模式的基本结构

1.1 抽象类

java
abstract class Game {
    // 模板方法：定义算法的骨架
    final void playGame() {
        initialize();
        startPlay();
        endPlay();
    }
    // 抽象方法：子类必须实现
    abstract void initialize();
    abstract void startPlay();
    abstract void endPlay();
    // 具体方法：子类可以重写
    void showResult() {
        System.out.println("游戏结束");
    }
}

1.2 子类

java
class Cricket extends Game {
    @Override
    void initialize() {
        System.out.println("Cricket 游戏初始化");
    }
    @Override
    void startPlay() {
        System.out.println("Cricket 游戏开始");
    }
    @Override
    void endPlay() {
        System.out.println("Cricket 游戏结束");
    }
}
class Football extends Game {
    @Override
    void initialize() {
        System.out.println("Football 游戏初始化");
    }
    @Override
    void startPlay() {
        System.out.println("Football 游戏开始");
    }
    @Override
    void endPlay() {
        System.out.println("Football 游戏结束");
    }
}

1.3 使用示例

java
public class Main {
    public static void main(String[] args) {
        Game cricket = new Cricket();
        cricket.playGame();
        Game football = new Football();
        football.playGame();
    }
}

> 输出：

text
Cricket 游戏初始化
Cricket 游戏开始
Cricket 游戏结束
游戏结束
Football 游戏初始化
Football 游戏开始
Football 游戏结束
游戏结束

接口 - 认识接口

接口 - 认识接口

在 Java 中，接口（Interface） 是一种引用类型，它定义了一组抽象方法，这些方法必须由实现该接口的类来实现。接口是面向对象编程的重要特性之一，它提供了多态性和代码的解耦。

1. 接口的基本语法

1.1 定义接口

java
public interface Flyable {
    void fly();
}

> 说明：

  • interface 关键字用于定义接口
  • 接口中的方法默认是 public abstract 的
  • 接口不能包含实例变量

1.2 实现接口

java
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}

> 说明：

  • implements 关键字用于实现接口
  • 实现类必须实现接口中的所有方法

2. 接口的特点

2.1 多态性

接口支持多态性，允许一个接口引用指向实现该接口的类的实例。

java
public interface Flyable {
    void fly();
}
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}
public class Plane implements Flyable {
    @Override
    public void fly() {
        System.out.println("飞机在飞");
    }
}
public class Main {
    public static void main(String[] args) {
        Flyable f1 = new Bird();
        Flyable f2 = new Plane();
        f1.fly(); // 输出: 鸟在飞
        f2.fly(); // 输出: 飞机在飞
    }
}

> 说明：

  • Flyable 接口定义了 fly() 方法
  • Bird 和 Plane 实现了 fly() 方法
  • 通过多态性，统一调用 fly() 方法

2.2 解耦

接口实现了代码的解耦，使得类之间的依赖关系更加松散。

java
public interface Flyable {
    void fly();
}
public class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}

    }
}

        f1.fly(); // 输出: 鸟在飞

    }
}

> 说明：

  • Flyable 接口定义了 fly() 方法
  • Bird 和 Plane 实现了 fly() 方法
  • 通过多态性，统一调用 fly() 方法

2.3 多重实现

一个类可以实现多个接口。

java
public interface Flyable {
    void fly();
}
public interface Swimmable {
    void swim();
}
public class Duck implements Flyable, Swimmable {
    @Override
    public void fly() {
        System.out.println("鸭子在飞");
    }
    @Override
    public void swim() {
        System.out.println("鸭子在游泳");
    }
}

> 说明：

  • Duck 类实现了 Flyable 和 Swimmable 接口
  • Duck 类必须实现两个接口中的所有方法

3.1 定义行为规范

在 Java 8 中，接口（Interface）得到了重大增强，引入了 默认方法（Default Methods）、静态方法（Static Methods） 和 私有方法（Private Methods），这些新特性使得接口更加灵活和强大。

  • 实现类可以重写 takeOff() 方法，也可以不重写

java

    }
}

> 说明：

  • displayInfo() 是静态方法
  • 可以通过 Flyable.displayInfo() 调用
