1. 48

继承 - 认识继承

在 Java 中，继承（Inheritance） 是面向对象编程的三大特性之一，它允许一个类（子类）继承另一个类（父类）的属性和方法，实现代码的复用和扩展。

1. 什么是继承？

继承是一种“is-a”关系，表示子类是父类的一种特殊类型。通过继承，子类可以复用父类的代码，同时可以添加新的功能或修改已有功能。

1.1 继承的语法

java
class 子类名 extends 父类名 {
    // 子类的属性和方法
}

1.2 示例

java
class Animal {
    String name;
    int age;
    void eat() {
        System.out.println("动物在吃东西");
    }
    void sleep() {
        System.out.println("动物在睡觉");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}

> 说明：

  • Dog 是 Animal 的子类
  • Dog 继承了 Animal 的 name、age、eat()、sleep() 方法
  • Dog 可以添加自己的方法 bark()

2. 继承的特点

特点说明
代码复用子类复用父类的代码，避免重复
扩展性子类可以添加新功能
多态性同一个接口可以有不同的实现方式
单继承Java 不支持多继承，一个类只能继承一个父类

    private String name; // 私有，子类不能访问
    protected int age; // 受保护，子类可以访问
    public String color; // 公共，任何地方都可以访问
}

4. 继承的使用场景

4.1 代码复用

java
class Person {
    String name;

    void eat() {
        System.out.println("人在吃东西");
    }
    void sleep() {
        System.out.println("人在睡觉");
    }
}
class Student extends Person {
    String school;
    void study() {
        System.out.println("学生在学习");
    }
}

> 说明：

  • Student 继承了 Person 的 name、age、eat()、sleep() 方法
  • Student 添加了自己的属性 school 和方法 study()

4.2 扩展功能

java
class Vehicle {
    String brand;

    void start() {
        System.out.println("车辆启动");
    }

        System.out.println("车辆停止");
    }
}
class Car extends Vehicle {
    int doors;
    void openDoors() {
        System.out.println("车门打开");
    }
}

> 说明：

  • Car 继承了 Vehicle 的 brand、speed、start()、stop() 方法
  • Car 添加了自己的属性 doors 和方法 openDoors()

5. 继承的注意事项

5.1 构造方法

  • 子类构造方法必须调用父类构造方法
  • 使用 super() 调用父类构造方法
java
class Animal {
    String name;
    int age;
    Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
class Dog extends Animal {
    Dog(String name, int age) {
        super(name, age); // 调用父类构造方法
    }
}

5.2 方法重写

  • 子类可以重写父类的方法，实现不同的行为
  • 使用 @Override 注解
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}

2. 49

继承 - 权限修饰符

在 Java 中，权限修饰符（Access Modifiers）用于控制类、属性、方法的访问范围。它们在继承中起着至关重要的作用，决定了子类能否访问父类的成员。

1. 权限修饰符的种类

Java 提供了四种权限修饰符，按访问权限从大到小排列：

修饰符说明访问范围
public公共的任何地方都可以访问
protected受保护的同包 + 子类可以访问
default（无修饰符）默认的同包可以访问
private私有的仅限本类访问

2. 权限修饰符的访问规则

2.1 访问权限表

修饰符同类同包子类其他
private✅❌❌❌
default✅✅✅❌
protected✅✅✅✅
public✅✅✅✅

3. 权限修饰符在继承中的应用

3.1 private 修饰符

  • 子类不能访问父类的 private 成员
  • private 成员仅限本类访问
java
class Animal {
    private String name; // 私有成员
    protected int age; // 受保护成员
    public String color; // 公共成员
}
class Dog extends Animal {
    void showInfo() {
        // System.out.println(name); // ❌ 编译错误：不能访问 private 成员
        System.out.println(age); // ✅ 可以访问 protected 成员
        System.out.println(color); // ✅ 可以访问 public 成员
    }
}

3.2 protected 修饰符

  • 子类可以访问父类的 protected 成员
  • protected 成员在不同包中也可以被子类访问
java
package com.example.animal;
public class Animal {
    protected String name;
    protected int age;
}
package com.example.dog;
import com.example.animal.Animal;
public class Dog extends Animal {
    void showInfo() {
        System.out.println("名字: " + name);
        System.out.println("年龄: " + age);
    }
}

3.3 default 修饰符

  • 同包中的类可以访问
  • 不同包中的子类不能访问
java
package com.example.animal;
class Animal {
    String name; // 默认修饰符
    int age; // 默认修饰符
}
package com.example.dog;
import com.example.animal.Animal;
public class Dog extends Animal {
    void showInfo() {
        System.out.println("名字: " + name); // ✅ 同包可以访问
        System.out.println("年龄: " + age); // ✅ 同包可以访问
    }
}

3.4 public 修饰符

  • 任何地方都可以访问
  • 子类可以访问父类的 public 成员
java
class Animal {
    public String name;
    public int age;
}
class Dog extends Animal {
    void showInfo() {
        System.out.println("名字: " + name);
        System.out.println("年龄: " + age);
    }
}

4. 权限修饰符的最佳实践

4.1 封装原则

  • 尽量使用 private 修饰成员变量
  • 提供 public 方法访问和修改私有成员
java
class Person {
    private String name;
    private int age;
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
    public void setAge(int age) {
        if (age > 0 && age < 150) {
            this.age = age;
        } else {
            System.out.println("年龄不合法");
        }
    }
    public int getAge() {
        return age;
    }
}

4.2 继承中的权限

  • 子类应尽量使用 protected 或 public 修饰符
  • 避免使用 private 修饰符，除非是内部实现细节

5. 总结

修饰符说明适用场景
private仅限本类访问实现细节
default同包访问包内类共享
protected同包 + 子类访问继承场景
public任何地方访问公共接口

> 权限修饰符 = 控制访问范围的工具

> 💡 建议： > - 在实际项目中，优先使用 private 封装数据 > - 在继承中，使用 protected 或 public 提供访问接口 > - 在测试中，使用 Mockito 模拟权限控制

3. 50

继承后的特点

在 Java 中，继承（Inheritance）是面向对象编程的核心特性之一。通过继承，子类可以复用父类的属性和方法，同时可以扩展或修改父类的功能。继承后，子类与父类之间形成了一种“is-a”的关系，即子类是父类的一种特殊类型。

1. 继承的基本特点

1.1 代码复用

继承允许子类复用父类的代码，避免重复编写相同的逻辑。

java
class Animal {
    String name;
    int age;
    void eat() {
        System.out.println("动物在吃东西");
    }
    void sleep() {
        System.out.println("动物在睡觉");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}

> 说明：

  • Dog 类继承了 Animal 类的 name、age、eat()、sleep() 方法
  • Dog 类可以使用这些继承的成员，无需重新定义

1.2 扩展功能

子类可以在继承的基础上添加新的属性和方法，实现功能扩展。

java
class Vehicle {
    String brand;
    int speed;
    void start() {
        System.out.println("车辆启动");
    }
    void stop() {
        System.out.println("车辆停止");
    }
}
class Car extends Vehicle {
    int doors;
    void openDoors() {
        System.out.println("车门打开");
    }
}

> 说明：

  • Car 类继承了 Vehicle 类的 brand、speed、start()、stop() 方法
  • Car 类添加了自己的属性 doors 和方法 openDoors()

2. 继承的访问控制

2.1 访问权限

  • 子类可以访问父类的 protected 和 public 成员
  • 子类不能访问父类的 private 成员
java
class Animal {
    private String name;
    protected int age;
    public String color;
}
class Dog extends Animal {
    void showInfo() {
        // System.out.println(name); // ❌ 编译错误：不能访问 private 成员
        System.out.println(age); // ✅ 可以访问 protected 成员
        System.out.println(color); // ✅ 可以访问 public 成员
    }
}

3. 继承的构造方法

3.1 构造方法的调用

  • 子类构造方法必须调用父类构造方法
  • 使用 super() 调用父类构造方法
java
class Animal {
    String name;
    int age;
    Animal(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
class Dog extends Animal {
    Dog(String name, int age) {
        super(name, age); // 调用父类构造方法
    }
}

> 说明：

  • super() 必须是子类构造方法的第一条语句
  • 如果父类没有无参构造方法，子类必须显式调用父类的构造方法

4. 继承的多态性

4.1 方法重写

子类可以重写父类的方法，实现不同的行为。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}

> 说明：

  • @Override 注解用于标记方法重写
  • 重写后，子类对象调用该方法时，执行子类的方法

4.2 多态性

多态性允许同一个接口有不同的实现方式。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("喵喵");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        a1.makeSound(); // 输出: 汪汪
        a2.makeSound(); // 输出: 喵喵
    }
}

> 说明：

  • Animal 类型的引用可以指向 Dog 或 Cat 对象
  • 调用 makeSound() 方法时，执行的是实际对象的方法

5. 继承的注意事项

5.1 单继承

  • Java 不支持多继承，一个类只能继承一个父类
  • 但可以实现多个接口
java
class Animal {
    void eat() {
        System.out.println("动物在吃东西");
    }
}
interface Flyable {
    void fly();
}
class Bird extends Animal implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}

5.2 构造方法的顺序

  • 构造方法的执行顺序是：父类构造方法 → 子类构造方法
  • 父类构造方法必须先执行
java
class Animal {
    Animal() {
        System.out.println("父类构造方法");
    }
}
class Dog extends Animal {
    Dog() {
        System.out.println("子类构造方法");
    }
}
public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
    }
}

> 输出：

text
父类构造方法
子类构造方法

4. 51

继承 - 方法重写 - 应用场景：toString() 方法

在 Java 中，toString() 方法是 Object 类提供的一个方法，用于返回对象的字符串表示。通过继承和方法重写，我们可以为自定义类提供更清晰、更有意义的字符串表示，方便调试和日志输出。

1. toString() 方法的作用

  • 默认行为：Object 类的 toString() 返回格式为 类名@哈希码
  • 重写目的：提供更直观、可读性强的字符串表示

1.1 默认 toString()

java
class Person {
    String name;
    int age;
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
public class Main {
    public static void main(String[] args) {
        Person p = new Person("张三", 20);
        System.out.println(p); // 输出: Person@1b6d3586
    }
}

> 说明：输出的是对象的内存地址，不直观。

2. 重写 toString() 方法

2.1 重写方式

java
class Person {
    String name;
    int age;
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    @Override
    public String toString() {
        return "Person{" +
                "name='" + name + '\'' +
                ", age=" + age +
                '}';
    }
}

2.2 使用示例

java
public class Main {
    public static void main(String[] args) {
        Person p = new Person("张三", 20);
        System.out.println(p); // 输出: Person{name='张三', age=20}
    }
}

> 说明：输出的是自定义的字符串表示，更直观。

3. toString() 方法的应用场景

3.1 调试和日志

  • 在调试时，输出对象信息
  • 在日志中记录对象状态
java
public class Logger {
    public static void log(Object obj) {
        System.out.println("日志信息: " + obj.toString());
    }
    public static void main(String[] args) {
        Person p = new Person("张三", 20);
        log(p); // 输出: 日志信息: Person{name='张三', age=20}
    }
}

3.2 输出集合中的对象

  • 集合类（如 List、Set）在打印时会调用对象的 toString() 方法
java
import java.util.ArrayList;
import java.util.List;
public class Main {
    public static void main(String[] args) {
        List<Person> list = new ArrayList<>();
        list.add(new Person("张三", 20));
        list.add(new Person("李四", 25));
        System.out.println(list); // 输出: [Person{name='张三', age=20}, Person{name='李四', age=25}]
    }
}

3.3 自定义对象的字符串表示

  • 提供更清晰的输出格式
  • 便于用户理解和使用
java
class Book {
    String title;
    String author;
    double price;
    Book(String title, String author, double price) {
        this.title = title;
        this.author = author;
        this.price = price;
    }
    @Override
    public String toString() {
        return "Book{" +
                "title='" + title + '\'' +
                ", author='" + author + '\'' +
                ", price=" + price +
                '}';
    }
}
public class Main {
    public static void main(String[] args) {
        Book b = new Book("Java编程思想", "Bruce Eckel", 99.0);
        System.out.println(b); // 输出: Book{title='Java编程思想', author='Bruce Eckel', price=99.0}
    }
}

4. toString() 方法的最佳实践

4.1 使用 @Override 注解

  • 明确表示重写父类方法
  • 避免拼写错误
java
@Override
public String toString() {
    return "Person{" +
            "name='" + name + '\'' +
            ", age=" + age +
            '}';
}

4.2 使用 StringBuilder 提高性能

  • 在 toString() 方法中，使用 StringBuilder 提高性能
java
@Override
public String toString() {
    StringBuilder sb = new StringBuilder();
    sb.append("Person{");
    sb.append("name='").append(name).append('\'');
    sb.append(", age=").append(age);
    sb.append('}');
    return sb.toString();
}

4.3 避免无限递归

  • 如果对象包含对自身的引用，可能导致栈溢出
java
class Circle {
    double radius;
    Circle(double radius) {
        this.radius = radius;
    }
    @Override
    public String toString() {
        return "Circle{" +
                "radius=" + radius +
                '}';
    }
}

> 说明：避免在 toString() 中调用自身的方法

5. 总结

特点说明
默认行为返回类名@哈希码
重写目的提供更清晰的字符串表示
应用场景调试、日志、集合输出
最佳实践使用 @Override、StringBuilder

> toString() = 对象的字符串表示

> 💡 建议： > - 在自定义类中重写 toString() 方法 > - 使用 StringBuilder 提高性能 > - 在调试和日志中使用 toString() 方法

5. 52

继承 - 子类构造器的特点与应用场景

在 Java 中，构造器（Constructor）用于初始化对象。当一个类继承另一个类时，子类的构造器必须调用父类的构造器，以确保父类的初始化逻辑被执行。理解子类构造器的特点对于正确使用继承至关重要。

1. 子类构造器的特点

1.1 构造器的调用顺序

  • 父类构造器先执行：子类构造器的第一条语句必须是 super() 或 this()，否则编译器会自动插入 super()。
  • 子类构造器后执行：父类构造器执行完毕后，子类构造器继续执行。
java
class Animal {
    String name;
    int age;
    Animal(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println("父类构造器执行");
    }
}
class Dog extends Animal {
    String breed;
    Dog(String name, int age, String breed) {
        super(name, age); // 调用父类构造器
        this.breed = breed;
        System.out.println("子类构造器执行");
    }
}

> 输出：

text

  • 隐式调用父类构造器：如果子类构造器没有显式调用 super()，编译器会自动插入 super()，前提是父类有无参构造器
java

    int age;

    }

class Dog extends Animal {
    String breed;

    }
}

  • 如果父类没有无参构造器，子类必须显式调用父类的有参构造器
  • 如果父类有无参构造器，子类可以不显式调用 super()，编译器会自动插入

继承 - 构造器用 this() 调用兄弟构造器

在 Java 中，this() 是一个特殊的语法，用于在同一个类中调用其他构造器。它通常用于构造器重载，避免重复代码。当一个类有多个构造器时，可以使用 this() 在一个构造器中调用另一个构造器，从而实现代码复用。

1. this() 的基本语法

java
this(参数列表);
  • this() 必须是构造器的第一条语句
  • this() 用于调用本类中的其他构造器
  • this() 不能在 static 方法或 static 块中使用

2. this() 的使用场景

2.1 构造器重载

当一个类有多个构造器时，可以使用 this() 调用其他构造器，避免重复代码。

java
class Person {
    String name;
    int age;
    String gender;
    // 无参构造器
    Person() {
        this("张三", 20, "男"); // 调用有参构造器
    }
    // 有参构造器
    Person(String name, int age, String gender) {
        this.name = name;
        this.age = age;
        this.gender = gender;
        System.out.println("有参构造器执行");
    }
}

> 说明：

  • this("张三", 20, "男") 调用 Person(String name, int age, String gender) 构造器
  • this() 必须是构造器的第一条语句

2.2 构造器链

this() 可以形成构造器链，从一个构造器调用另一个构造器，最终调用最底层的构造器。

java

    int age;
    String gender;

        this(name, age, "男"); // 调用三参构造器

    // 三参构造器
    Person(String name, int age, String gender) {
        this.name = name;
        this.age = age;
        this.gender = gender;
        System.out.println("三参构造器执行");
    }
}

> 说明：

  • this("张三", 20, "男") 调用 Person(String name, int age, String gender) 构造器
  • this(name, age) 调用 Person(String name, int age, String gender) 构造器

3. this() 的注意事项

3.1 this() 必须是构造器的第一条语句

java
class Person {
    String name;
    int age;
    Person(String name, int age) {
        System.out.println("执行其他逻辑");

> 说明：this() 必须是构造器的第一条语句，否则编译器会报错。

3.2 this() 不能调用 super()

java

> 说明：this() 用于调用本类的其他构造器，super() 用于调用父类的构造器。

3.3 this() 不能在 static 方法中使用

    int age;

        this.name = name;
        this.age = age;
        System.out.println("有参构造器执行");
    }

> 说明：this() 只能在实例构造器中使用，不能在 static 方法或 static 块中使用。

多态 - 认识多态

在 Java 中，多态（Polymorphism） 是面向对象编程的三大特性之一，它允许同一个接口或方法在不同对象上有不同的实现方式。多态的核心思想是“一个接口，多种实现”，它提高了程序的灵活性和可扩展性。

1. 什么是多态？

多态是指同一个方法在不同对象上表现出不同的行为。它通过继承和方法重写实现，使得程序可以在运行时根据对象的实际类型选择调用哪个方法。

1.1 多态的实现方式

  • 继承：子类继承父类
  • 方法重写：子类重写父类的方法
  • 向上转型：父类引用指向子类对象

2. 多态的基本语法

2.1 父类和子类

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("喵喵");
    }
}

2.2 向上转型

java
public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog(); // 向上转型
        Animal a2 = new Cat(); // 向上转型
        a1.makeSound(); // 输出: 汪汪
        a2.makeSound(); // 输出: 喵喵
    }
}

> 说明：

  • Animal a1 = new Dog();：父类引用指向子类对象
  • 调用 makeSound() 方法时，执行的是子类的方法

3. 多态的特点

3.1 动态绑定

  • 多态的实现依赖于动态绑定（Runtime Binding）
  • 在运行时根据对象的实际类型决定调用哪个方法
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}

    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.makeSound(); // 输出: 汪汪
    }
}

> 说明：

  • a 是 Animal 类型的引用
  • 但 a 实际指向 Dog 对象
  • 调用 makeSound() 方法时，执行的是 Dog 类的方法

3.2 方法重写

  • 子类可以重写父类的方法，实现不同的行为
  • 使用 @Override 注解标记方法重写
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}

> 说明：

  • @Override 注解用于标记方法重写
  • 提高代码可读性和安全性

多态 - 多态的好处和存在的问题

多态是面向对象编程的核心特性之一，它通过继承和方法重写，实现了“一个接口，多种实现”的灵活性。然而，多态在带来便利的同时，也存在一些潜在的问题。本节将详细探讨多态的好处和存在的问题。

1. 多态的好处

1.1 提高代码的可扩展性

多态允许我们在不修改现有代码的情况下，添加新的子类。这符合“开闭原则”（Open-Closed Principle），即对扩展开放，对修改关闭。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("喵喵");
    }
}
public class Main {
    public static void main(String[] args) {
        List<Animal> animals = new ArrayList<>();
        animals.add(new Dog());
        animals.add(new Cat());
        for (Animal animal : animals) {
            animal.makeSound(); // 输出: 汪汪, 喵喵
        }
    }
}

> 说明：

  • 添加新的子类（如 Bird）时，无需修改 main 方法
  • 代码具有良好的可扩展性

1.2 提高代码的可维护性

多态使得代码更加模块化，每个类负责自己的行为，降低了耦合度。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");

}
class Dog extends Animal {

    }
}

> 说明：

  • 每个类只关注自己的行为
  • 修改一个类不会影响其他类

1.3 提高代码的灵活性

多态允许我们在运行时根据对象的实际类型选择调用哪个方法，提高了程序的灵活性。

java

  • 但 a 实际指向 Dog 对象
  • 调用 makeSound() 方法时，执行的是 Dog 类的方法

2.1 向下转型的风险

  • 如果类型不匹配，会抛出 ClassCastException

        Dog d = (Dog) a; // 向下转型
        d.bark(); // ✅ 可以访问
    }
}

> 说明：

  • a 是 Animal 类型的引用
  • d 是 Dog 类型的引用
  • d.bark() 可以访问 Dog 类中特有的方法

2.2 方法重写的影响

  • 子类重写父类的方法，可能会影响父类的逻辑
  • 需要确保重写的方法符合父类的契约
java
class Animal {
    void makeSound() {

    }
}

> 说明：

  • Dog 类重写了 makeSound() 方法
  • 父类的逻辑被覆盖

2.3 性能开销

  • 多态依赖于动态绑定，有一定的性能开销
  • 在性能敏感的场景中，需要权衡
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");

多态 - 多态下的类型转换问题

在 Java 中，多态允许父类引用指向子类对象，这为程序设计带来了极大的灵活性。然而，这种灵活性也带来了类型转换的问题。本节将详细探讨多态下的类型转换问题，包括向上转型、向下转型以及相关的注意事项。

1. 向上转型（Upcasting）

1.1 定义

向上转型是指将子类对象赋值给父类引用的过程。这是自动进行的，无需显式转换。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("汪汪");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog(); // 向上转型
        a.makeSound(); // 输出: 汪汪
    }
}

> 说明：

  • Animal a = new Dog();：将 Dog 对象赋值给 Animal 引用
  • a 是 Animal 类型的引用，但指向 Dog 对象
  • 调用 makeSound() 方法时，执行的是 Dog 类的方法

1.2 特点

  • 自动进行：无需显式转换
  • 只能访问父类中定义的方法：不能访问子类中特有的方法
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        a.makeSound(); // ✅ 可以访问
        // a.bark(); // ❌ 编译错误：不能访问子类特有的方法
    }
}

> 说明：

  • a 是 Animal 类型的引用
  • 但 a 实际指向 Dog 对象
  • a.bark() 会编译错误，因为 Animal 类中没有 bark() 方法

2. 向下转型（Downcasting）

2.1 定义

向下转型是指将父类引用转换为子类引用的过程。这是强制进行的，需要显式转换。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        Dog d = (Dog) a; // 向下转型
        d.bark(); // ✅ 可以访问
    }
}

> 说明：

  • a 是 Animal 类型的引用
  • d 是 Dog 类型的引用
  • d.bark() 可以访问 Dog 类中特有的方法

2.2 特点

  • 强制进行：需要显式转换
  • 类型检查：在运行时检查类型是否匹配
  • 类型不匹配时抛出异常：如果类型不匹配，会抛出 ClassCastException
java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    void meow() {
        System.out.println("喵喵");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        Dog d = (Dog) a; // ✅ 类型匹配
        d.bark(); // ✅ 可以访问
        Animal b = new Cat();
        // Dog d2 = (Dog) b; // ❌ 类型不匹配，抛出 ClassCastException
    }
}

> 说明：

  • a 是 Dog 对象，可以向下转型为 Dog
  • b 是 Cat 对象，不能向下转型为 Dog，会抛出 ClassCastException

3. 类型检查

3.1 使用 instanceof 操作符

在进行向下转型之前，可以使用 instanceof 操作符检查类型是否匹配。

java
class Animal {
    void makeSound() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    void bark() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    void meow() {
        System.out.println("喵喵");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a = new Dog();
        if (a instanceof Dog) {
            Dog d = (Dog) a;
            d.bark(); // ✅ 可以访问
        } else {
            System.out.println("不是 Dog 对象");
        }
    }
}

> 说明：

  • instanceof 操作符用于检查对象是否是某个类的实例
  • 如果类型匹配，可以安全地进行向下转型

3.2 使用 instanceof 的好处

  • 避免 ClassCastException：在进行向下转型之前检查类型
  • 提高代码安全性：确保类型匹配

> 类型转换 = 多态的灵活性与安全性的平衡

import java.util.Scanner;
public class PaymentSystem {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("请输入支付金额: ");
        double amount = scanner.nextDouble();
        System.out.println("请选择支付方式:");
        System.out.println("1. 支付宝");
        System.out.println("2. 微信支付");
        System.out.println("3. 银行卡支付");
        System.out.print("请输入选择: ");
        int choice = scanner.nextInt();
        Payment payment = null;

            case 2:
                payment = new WeChatPay(amount);
                break;
            case 3:
                payment = new BankCardPay(amount);
                break;
            default:
                System.out.println("无效选择");
                return;
        }
        payment.pay(); // 调用支付方法
        scanner.close();
    }
}
