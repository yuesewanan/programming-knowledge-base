面向对象编程入门

面向对象编程入门

面向对象编程（Object-Oriented Programming，简称 OOP）是 Java 的核心编程范式，它将现实世界中的事物抽象为“对象”，通过封装、继承、多态等特性实现代码的模块化、可维护性和可扩展性。

1. 什么是面向对象编程？

面向对象编程是一种以“对象”为核心的编程思想，它将数据和操作数据的方法封装在一起，形成一个独立的单元。

1.1 核心思想

  • 万物皆对象：现实世界中的事物都可以抽象为对象
  • 对象 = 属性 + 方法：对象包含数据（属性）和行为（方法）
  • 程序 = 对象之间的交互：通过调用对象的方法实现功能

2. 面向对象的四大特性

特性说明
封装将数据和方法封装在类中，隐藏内部实现细节
继承子类继承父类的属性和方法，实现代码复用
多态同一个接口可以有不同的实现方式，提高灵活性
抽象抽象出共性，定义接口或抽象类，隐藏复杂性

3. 类与对象

3.1 什么是类？

类是对象的模板或蓝图，用于描述对象的属性和行为。

3.2 什么是对象？

对象是类的实例，是具体的实体。

3.3 示例

java
// 定义一个类
class Student {
    // 属性（成员变量）
    String name;
    int age;
    double score;
    // 方法（成员方法）
    void study() {
        System.out.println(name + " 正在学习");
    }
    void showInfo() {
        System.out.println("姓名: " + name + ", 年龄: " + age + ", 成绩: " + score);
    }
}
// 创建对象
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student(); // 创建对象
        s1.name = "张三";
        s1.age = 20;
        s1.score = 90.5;
        s1.showInfo(); // 输出: 姓名: 张三, 年龄: 20, 成绩: 90.5
        s1.study(); // 输出: 张三 正在学习
    }
}

4. 封装

封装是面向对象编程的第一特性，它将数据和方法封装在类中，隐藏内部实现细节。

4.1 封装的好处

  • 提高安全性：防止外部直接修改内部数据
  • 提高可维护性：修改内部实现不影响外部调用
  • 提高灵活性：可以通过方法控制数据访问

4.2 示例

java
class Student {
    private String name; // 私有属性
    private int age;
    // 提供公共方法访问私有属性
    public void setName(String name) {
        this.name = name;
    }
    public String getName() {
        return name;
    }
    public void setAge(int age) {
        if (age > 0 && age < 150) {
            this.age = age;
        } else {
            System.out.println("年龄不合法");
        }
    }
    public int getAge() {
        return age;
    }
}

5. 继承

继承是面向对象编程的第二特性，它允许一个类继承另一个类的属性和方法，实现代码复用。

5.1 继承的语法

java
class Parent {
    String name;
    void showName() {
        System.out.println("父类姓名: " + name);
    }
}
class Child extends Parent {
    int age;
    void showAge() {
        System.out.println("子类年龄: " + age);
    }
}

5.2 示例

java
public class Main {
    public static void main(String[] args) {
        Child c = new Child();
        c.name = "张三";
        c.age = 20;
        c.showName(); // 输出: 父类姓名: 张三
        c.showAge(); // 输出: 子类年龄: 20
    }
}

6. 多态

多态是面向对象编程的第三特性，它允许同一个接口有不同的实现方式，提高灵活性。

6.1 多态的实现方式

  • 方法重写：子类重写父类的方法
  • 方法重载：同一个类中定义多个同名方法

6.2 示例

java
class Animal {
    void speak() {
        System.out.println("动物叫");
    }
}
class Dog extends Animal {
    @Override
    void speak() {
        System.out.println("汪汪");
    }
}
class Cat extends Animal {
    @Override
    void speak() {
        System.out.println("喵喵");
    }
}
public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();
        a1.speak(); // 输出: 汪汪
        a2.speak(); // 输出: 喵喵
    }
}

7. 抽象

抽象是面向对象编程的第四特性，它通过定义接口或抽象类，隐藏复杂性，只暴露必要的接口。

7.1 抽象类

java
abstract class Shape {
    abstract double area(); // 抽象方法
}
class Circle extends Shape {
    double radius;
    Circle(double r) {
        radius = r;
    }
    @Override
    double area() {
        return Math.PI * radius * radius;
    }
}

7.2 接口

java
interface Flyable {
    void fly();
}
class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("鸟在飞");
    }
}

8. 总结

特性说明
封装数据和方法封装在类中
继承子类继承父类的属性和方法
多态同一个接口有不同的实现方式
抽象定义接口或抽象类，隐藏复杂性

> 面向对象编程 = 面向现实世界的编程方式

> 💡 建议： > - 在实际项目中，优先使用封装保护数据 > - 在复杂逻辑中，考虑使用继承和多态 > - 在抽象层面，使用接口定义行为

构造器

构造器

在 Java 面向对象编程中，构造器（Constructor） 是一种特殊的成员方法，用于在创建对象时初始化对象的状态。构造器与类同名，没有返回类型（甚至没有 void），并且在对象实例化时自动调用。

构造器的基本语法

java
public class Person {
    private String name;
    private int age;
    // 无参构造器
    public Person() {
        this.name = "Unknown";
        this.age = 0;
    }
    // 带参构造器
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
}

构造器的特性

特性说明
名称必须与类名一致与类名完全相同，区分大小写
无返回类型包括 void 也不允许
自动调用在使用 new 关键字创建对象时自动调用
可重载一个类可以有多个构造器，参数列表不同
默认构造器如果类中没有定义任何构造器，编译器会自动提供一个无参构造器

构造器的重载

Java 支持构造器重载，即一个类中可以有多个构造器，只要它们的参数列表不同。

java
public class Student {
    private String name;
    private int age;
    private String major;
    // 无参构造器
    public Student() {}
    // 带参构造器
    public Student(String name) {
        this.name = name;
    }
    // 带参构造器
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
    // 带参构造器
    public Student(String name, int age, String major) {
        this.name = name;
        this.age = age;
        this.major = major;
    }
}

this 关键字在构造器中的使用

this 关键字可以用于引用当前对象的实例变量，也可以用于调用类中的其他构造器。

java
public class Rectangle {
    private double width;
    private double height;
    // 默认构造器
    public Rectangle() {
        this(1.0, 1.0); // 调用带参构造器
    }
    // 带参构造器
    public Rectangle(double width, double height) {
        this.width = width;
        this.height = height;
    }
}

构造器的访问控制

构造器可以使用 public、private、protected 或默认访问修饰符。

  • public：任何类都可以访问
  • private：只能在本类中访问，常用于单例模式
  • protected：子类和同包类可以访问
  • 默认（包私有）：同包类可以访问
java
public class Singleton {
    private static Singleton instance;
    private Singleton() {} // 私有构造器
    public static Singleton getInstance() {
        if (instance == null) {
            instance = new Singleton();
        }
        return instance;
    }
}

构造器的执行顺序

当创建一个对象时，构造器的执行顺序如下：

1. 父类的无参构造器（如果父类没有无参构造器，则必须显式调用父类的构造器） 2. 子类的构造器

java
class Animal {
    public Animal() {
        System.out.println("Animal 构造器");
    }
}
class Dog extends Animal {
    public Dog() {
        System.out.println("Dog 构造器");
    }
}
// 输出：
// Animal 构造器
// Dog 构造器

总结

  • 构造器用于初始化对象
  • 可以重载，但参数列表必须不同
  • this 可以用于调用其他构造器或引用实例变量
  • 构造器可以控制访问权限，影响对象的创建方式
  • 构造器的执行顺序遵循继承链
构造器是 Java 面向对象编程中非常重要的概念，合理使用构造器可以提高代码的可读性和可维护性。

this关键字

this关键字

在 Java 面向对象编程中，this 关键字 是一个非常重要的引用，它代表当前对象的引用，即调用该方法或构造器的对象本身。this 可以用于访问当前对象的实例变量、调用当前对象的方法，或在构造器中调用其他构造器。

1. this 用于访问实例变量

当类的局部变量与实例变量同名时，this 可以用于明确指定访问的是实例变量，而不是局部变量。

java
public class Person {
    private String name;
    private int age;
    public Person(String name, int age) {
        this.name = name; // this.name 指的是实例变量
        this.age = age;   // this.age 指的是实例变量
    }
    public void setName(String name) {
        this.name = name; // 避免与参数 name 混淆
    }
    public void setAge(int age) {
        this.age = age;   // 避免与参数 age 混淆
    }
}

2. this 用于调用当前对象的方法

this 可以用于调用当前对象的其他方法，尤其在方法链中非常有用。

java
public class Calculator {
    private int result;
    public Calculator add(int x) {
        result += x;
        return this; // 返回当前对象，支持方法链
    }
    public Calculator multiply(int x) {
        result *= x;
        return this;
    }
    public int getResult() {
        return result;
    }
    // 使用示例
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        calc.add(5).multiply(3).add(2);
        System.out.println(calc.getResult()); // 输出 17
    }
}

3. this 用于在构造器中调用其他构造器

this() 可以在构造器中调用同一个类中的其他构造器，必须放在构造器的第一行。

java
public class Student {
    private String name;
    private int age;
    private String major;
    // 无参构造器
    public Student() {
        this("Unknown", 0, "Unknown"); // 调用带参构造器
    }
    // 带参构造器
    public Student(String name, int age) {
        this(name, age, "Computer Science"); // 调用三参构造器
    }
    // 三参构造器
    public Student(String name, int age, String major) {
        this.name = name;
        this.age = age;
        this.major = major;
    }
}

4. this 用于返回当前对象

在方法中返回 this 可以实现方法链（Method Chaining），提高代码的可读性和简洁性。

java
public class StringBuilderExample {
    private StringBuilder sb;
    public StringBuilderExample() {
        sb = new StringBuilder();
    }
    public StringBuilderExample append(String str) {
        sb.append(str);
        return this; // 返回当前对象
    }
    public StringBuilderExample append(int num) {
        sb.append(num);
        return this;
    }
    public String toString() {
        return sb.toString();
    }
    // 使用示例
    public static void main(String[] args) {
        StringBuilderExample builder = new StringBuilderExample();
        builder.append("Hello").append(" ").append("World").append(123);
        System.out.println(builder.toString()); // 输出: Hello World123
    }
}

5. this 用于传递当前对象作为参数

this 可以作为参数传递给其他方法或对象，用于将当前对象传递给其他类处理。

java
public class Employee {
    private String name;
    private double salary;
    public Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }
    public void register(EmployeeManager manager) {
        manager.addEmployee(this); // 将当前对象传递给管理器
    }
    // getter 和 setter 省略
}
public class EmployeeManager {
    private List<Employee> employees = new ArrayList<>();
    public void addEmployee(Employee employee) {
        employees.add(employee);
    }
    public void printAll() {
        for (Employee e : employees) {
            System.out.println(e.getName() + " - " + e.getSalary());
        }
    }
}

6. this 的访问权限

this 可以访问当前对象的所有成员，包括：

  • 公有（public）成员
  • 保护（protected）成员
  • 默认（包私有）成员
  • 私有（private）成员（仅限本类内部）

7. 注意事项

  • this 不能在静态方法或静态代码块中使用，因为静态成员属于类，而不是对象。
  • this() 必须是构造器的第一条语句。
  • this 不能用于访问静态变量或静态方法（虽然语法上允许，但不推荐，因为静态成员属于类，不是对象）。

总结

用途说明
访问实例变量避免与局部变量同名冲突
调用其他方法支持方法链，提高代码可读性
调用其他构造器构造器重载时使用，必须放在第一行
返回当前对象实现方法链
传递当前对象作为参数传递给其他方法或对象

this 关键字是 Java 面向对象编程中的核心概念之一，合理使用 this 可以使代码更加清晰、简洁、可维护。

封装

封装

在 Java 面向对象编程中，封装（Encapsulation） 是四大基本特性之一，指的是将对象的属性和行为封装在类中，隐藏内部实现细节，只暴露必要的接口供外部使用。封装的目的是提高代码的安全性、可维护性和可复用性。

1. 封装的核心思想

封装的本质是“隐藏实现，暴露接口”。通过访问修饰符（如 private、public 等）控制类成员的可见性，防止外部直接访问或修改对象的内部状态。

2. 封装的实现方式

2.1 使用访问修饰符

  • private：私有成员，只能在本类中访问
  • public：公有成员，任何地方都可以访问
  • protected：受保护成员，子类和同包类可以访问
  • 默认（包私有）：同包类可以访问
java
public class BankAccount {
    private String accountNumber; // 私有属性
    private double balance;       // 私有属性
    // 公有方法，用于访问和修改属性
    public String getAccountNumber() {
        return accountNumber;
    }
    public double getBalance() {
        return balance;
    }
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
}

2.2 Getter 和 Setter 方法

封装通常通过提供 getter 和 setter 方法来实现，允许外部安全地读取和修改属性。

java
public class Person {
    private String name;
    private int age;
    // Getter
    public String getName() {
        return name;
    }
    // Setter
    public void setName(String name) {
        if (name != null && !name.trim().isEmpty()) {
            this.name = name;
        }
    }
    // Getter
    public int getAge() {
        return age;
    }
    // Setter
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {
            this.age = age;
        }
    }
}

3. 封装的好处

好处说明
安全性防止外部直接修改对象状态，避免非法数据
可维护性修改内部实现不影响外部代码，只需修改 getter/setter
可复用性封装良好的类更容易被其他模块复用
灵活性可以在 getter/setter 中添加验证、日志、计算等逻辑

4. 封装的高级应用

4.1 数据验证

在 setter 中添加数据验证逻辑，确保数据合法性。

java
public class Product {
    private String name;
    private double price;
    public void setPrice(double price) {
        if (price >= 0) {
            this.price = price;
        } else {
            throw new IllegalArgumentException("价格不能为负数");
        }
    }
}

4.2 访问控制

通过封装，可以控制哪些属性对外可见，哪些不可见。

java
public class Car {
    private String brand;
    private int speed; // 不希望外部直接修改
    public String getBrand() {
        return brand;
    }
    public void setBrand(String brand) {
        this.brand = brand;
    }
    public int getSpeed() {
        return speed;
    }
    // 仅允许通过方法控制速度
    public void accelerate(int increment) {
        speed += increment;
    }
    public void brake(int decrement) {
        speed = Math.max(0, speed - decrement);
    }
}

5. 封装与设计模式

封装是许多设计模式的基础，例如：

  • 单例模式：通过私有构造器和静态方法控制对象创建
  • 工厂模式：封装对象创建过程
  • 观察者模式：封装状态变化通知机制

6. 封装的注意事项

  • 不要过度封装，导致接口过于复杂
  • 避免暴露不必要的方法或属性
  • 在 setter 中添加必要的业务逻辑（如验证、日志等）
  • 使用 final 修饰不可变对象，增强封装性

总结

封装是面向对象编程的基石之一。通过合理使用访问修饰符和提供安全的访问方法，可以有效保护对象的内部状态，提高代码的健壮性和可维护性。良好的封装设计是构建高质量 Java 程序的关键。

> 封装 = 私有化属性 + 公有化方法 + 数据验证 + 接口控制

在实际开发中，应遵循“最小暴露原则”，只暴露必要的接口，隐藏不必要的实现细节。

实体类

实体类

在 Java 开发中，实体类（Entity Class） 是用于表示现实世界中某个具体对象或数据结构的类。它通常对应数据库中的一张表，或系统中的一个业务对象，封装了对象的属性和基本操作。

实体类是面向对象编程中“类”概念的直接体现，广泛应用于 Web 开发、数据持久化、业务逻辑处理等场景。

1. 实体类的基本结构

一个典型的实体类包含：

  • 私有属性（private fields）：表示对象的属性
  • 公共的 getter 和 setter 方法：用于访问和修改属性
  • 构造器（可选）：用于初始化对象
  • toString() 方法（可选）：用于输出对象信息
  • equals() 和 hashCode() 方法（可选）：用于对象比较
java
public class User {
    private String id;
    private String username;
    private String email;
    private int age;
    // 无参构造器
    public User() {}
    // 带参构造器
    public User(String id, String username, String email, int age) {
        this.id = id;
        this.username = username;
        this.email = email;
        this.age = age;
    }
    // Getter 和 Setter
    public String getId() { return id; }
    public void setId(String id) { this.id = id; }
    public String getUsername() { return username; }

    public String toString() {

                ", age=" + age +
                '}';
    }
    // equals 和 hashCode（用于集合比较）

    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        User user = (User) o;
        return age == user.age &&
                Objects.equals(id, user.id) &&
                Objects.equals(username, user.username) &&
                Objects.equals(email, user.email);

static - 静态变量 - 应用场景

在 Java 中，static 关键字用于声明静态变量（也称类变量），它属于类本身，而不是类的实例。静态变量在类加载时初始化，所有实例共享同一份内存空间。

1. 静态变量的定义

java
public class Counter {
    private static int count = 0; // 静态变量，所有实例共享
    public Counter() {
        count++; // 每创建一个实例，计数器加1
    }
    public static int getCount() {
        return count;
    }
}

> 注意：静态变量必须在类加载时初始化，不能依赖实例变量。

静态方法的注意事项

在 Java 中，static 关键字用于定义静态成员，包括静态变量、静态方法和静态代码块。静态方法属于类本身，而不是类的实例，因此在使用时需要注意以下几点，以避免常见错误和性能问题。

1. 静态方法的访问限制

1.1 不能访问非静态成员

静态方法不能直接访问类的非静态成员（实例变量和实例方法）。

java
public class MyClass {
    private int instanceVar = 10; // 实例变量
    private static int staticVar = 20; // 静态变量
    public void instanceMethod() {
        System.out.println("实例方法");
    }
    public static void staticMethod() {
        // System.out.println(instanceVar); // ❌ 编译错误：不能直接访问非静态成员
        System.out.println(staticVar); // ✅ 可以访问静态成员
        // instanceMethod(); // ❌ 编译错误：不能直接调用实例方法
    }
}

> ✅ 解决方案：通过对象访问实例成员

java
public static void staticMethod() {
    MyClass obj = new MyClass();
    System.out.println(obj.instanceVar); // ✅ 通过对象访问
    obj.instanceMethod(); // ✅ 通过对象调用
}

2. 静态方法的调用方式

2.1 通过类名调用

java
MyClass.staticMethod(); // ✅ 推荐方式

2.2 通过对象调用

java
MyClass obj = new MyClass();
obj.staticMethod(); // ✅ 但不推荐，容易误解为实例方法

> ✅ 推荐使用类名调用，避免混淆。

3. 静态方法的生命周期

3.1 静态方法的生命周期

  • 静态方法在类加载时初始化
  • 静态方法的生命周期与类相同
  • 静态方法在类卸载时销毁

3.2 静态方法的内存管理

  • 静态方法存储在方法区（Method Area）
  • 静态方法的内存空间是共享的

4. 静态方法的线程安全

4.1 静态方法的线程安全

  • 静态方法本身是线程安全的，但其内部操作可能不是线程安全的
  • 如果静态方法修改了静态变量，需要考虑线程安全
#### 示例：非线程安全的静态方法

java
public class Counter {
    private static int count = 0;
    public static void increment() {
        count++; // ❌ 非线程安全
    }
    public static int getCount() {
        return count;
    }
}

> ✅ 解决方案：使用 synchronized 或 AtomicInteger

java
public class Counter {
    private static AtomicInteger count = new AtomicInteger(0);
    public static void increment() {
        count.incrementAndGet(); // ✅ 线程安全
    }
    public static int getCount() {
        return count.get();
    }
}

5.2 静态方法的性能劣势

  • 静态方法不能访问实例成员

6. 静态方法的最佳实践

6.2 不要滥用静态方法

  • 静态方法不适合复杂逻辑
  • 静态方法不适合需要状态管理的场景

6.3 避免过度使用

  • 静态方法不利于测试和扩展

        // 按类型筛选
        System.out.println("\n科幻电影：");
        filterMoviesByGenre(movies, "科幻");
    }
    // 按评分排序
    public static void sortMoviesByRating(Movie[] movies) {
        Arrays.sort(movies, (m1, m2) -> Double.compare(m2.getRating(), m1.getRating())); // 降序
    }
    // 按类型筛选
    public static void filterMoviesByGenre(Movie[] movies, String genre) {
        for (Movie movie : movies) {
            if (movie.getGenre().equals(genre)) {
                System.out.println(movie);
            }
        }
    }
}
