综合项目-智能家居系统

在本节中，我们将通过一个综合项目来实践 Java 的核心概念，构建一个简单的智能家居系统。该项目将结合面向对象编程（OOP）、接口、抽象类、多态性等知识点，帮助你巩固所学知识并提升实际开发能力。

项目目标

实现一个智能家居系统的控制平台，支持以下功能：

  • 控制灯光、空调、窗帘等设备
  • 设备可以是不同的类型（如智能灯、智能空调）
  • 支持通过统一接口进行操作
  • 实现设备状态的显示与切换

设计思路

├── AirConditioner (继承 Device)

> 说明：

  • Device 是一个抽象类，定义了所有设备的公共行为
  • Light、AirConditioner、Curtain 是具体的设备类
  • ControlCenter 是控制中心，负责管理所有设备

代码实现

抽象类 Device

java
public abstract class Device {
    protected String name;
    protected boolean isOn;
    public Device(String name) {
        this.name = name;
        this.isOn = false;
    }
    // 抽象方法：打开设备
    public abstract void turnOn();
    // 抽象方法：关闭设备
    public abstract void turnOff();
    // 普通方法：获取设备状态
    public void displayStatus() {
        System.out.println(name + " 当前状态：" + (isOn ? "开启" : "关闭"));
    }
}

> 说明：

  • Device 定义了所有设备的公共属性和行为
  • turnOn() 和 turnOff() 是抽象方法，由子类实现
  • displayStatus() 是普通方法，用于显示设备状态

控制中心 ControlCenter

import java.util.ArrayList;
import java.util.List;
public class ControlCenter {
    private List<Device> devices;
    public ControlCenter() {
        this.devices = new ArrayList<>();
    }
    public void addDevice(Device device) {
        devices.add(device);
    }
    public void turnOnAll() {
        System.out.println("正在打开所有设备...");
        for (Device device : devices) {
            device.turnOn();
        }
    }
    public void turnOffAll() {
        System.out.println("正在关闭所有设备...");
        for (Device device : devices) {
            device.turnOff();
        }
    }
    public void displayAllStatus() {
        System.out.println("设备状态列表：");
        for (Device device : devices) {
            device.displayStatus();
        }
    }
}

类中的成分-代码块

在 Java 中，类的组成部分除了字段（成员变量）、方法、构造器之外，还有一种特殊的结构叫做 代码块（Code Block）。代码块用于在类加载或对象创建时执行特定的初始化逻辑，是 Java 中实现复杂初始化需求的重要机制。

1. 什么是代码块？

代码块是指在类中使用 {} 包裹的一段代码，它不属于任何方法，但可以在类中定义并执行。根据其位置和特性，代码块主要分为以下几种类型：

  • 静态代码块（Static Block）
  • 实例代码块（Instance Block）
  • 构造器代码块（Constructor Block）
> 注意：这里的“构造器代码块”并不是一个独立的语法结构，而是指构造器内部的代码逻辑。我们通常所说的代码块指的是静态代码块和实例代码块。

2. 静态代码块（Static Block）

2.1 定义方式

java
static {
    // 静态代码块中的代码
}

2.2 特点

  • 使用 static 关键字修饰
  • 在类加载时执行，且只执行一次
  • 优先于构造器执行
  • 可用于初始化静态变量或执行一次性的静态资源加载操作

2.3 示例

java
public class StaticBlockDemo {
    static int count = 0;
    static {
        System.out.println("静态代码块执行");
        count = 100;
    }
    public StaticBlockDemo() {
        System.out.println("构造器执行");
    }
    public static void main(String[] args) {
        System.out.println("main 方法执行");
        new StaticBlockDemo();
        new StaticBlockDemo();
    }
}

2.4 输出结果

text
静态代码块执行
main 方法执行
构造器执行
构造器执行

> 说明：

  • 静态代码块在类加载时执行，且只执行一次
  • main 方法在静态代码块之后执行
  • 构造器在每次创建对象时执行

3. 实例代码块（Instance Block）

3.1 定义方式

java
{
    // 实例代码块中的代码
}

3.2 特点

  • 不使用 static 关键字修饰
  • 在每次创建对象时执行
  • 优先于构造器执行
  • 可用于初始化实例变量或执行每次对象创建时都需要执行的逻辑

3.3 示例

java
public class InstanceBlockDemo {
    int id;
    String name;
    {
        System.out.println("实例代码块执行");
        id = 1;
        name = "默认名称";
    }
    public InstanceBlockDemo() {
        System.out.println("无参构造器执行");
    }
    public InstanceBlockDemo(int id, String name) {
        this.id = id;
        this.name = name;
        System.out.println("有参构造器执行");
    }
    public static void main(String[] args) {
        System.out.println("main 方法执行");
        new InstanceBlockDemo();
        new InstanceBlockDemo(2, "张三");
    }
}

3.4 输出结果

text
main 方法执行
实例代码块执行
无参构造器执行
实例代码块执行
有参构造器执行

> 说明：

  • 实例代码块在每次创建对象时执行
  • 实例代码块在构造器之前执行
  • 实例代码块可以用于统一初始化多个构造器共用的逻辑

4. 代码块的执行顺序

当一个类被加载并创建对象时，代码块的执行顺序如下：

1. 静态代码块：类加载时执行，只执行一次 2. 实例代码块：每次创建对象时执行 3. 构造器：每次创建对象时执行

4.1 示例

java
public class BlockOrderDemo {
    static {
        System.out.println("静态代码块1");
    }
    {
        System.out.println("实例代码块1");
    }
    public BlockOrderDemo() {
        System.out.println("构造器1");
    }
    static {
        System.out.println("静态代码块2");
    }
    {
        System.out.println("实例代码块2");
    }
    public BlockOrderDemo(int i) {
        System.out.println("构造器2");
    }
    public static void main(String[] args) {
        System.out.println("main 方法执行");
        new BlockOrderDemo();
        new BlockOrderDemo(1);
    }
}

4.2 输出结果

text
静态代码块1
静态代码块2
main 方法执行
实例代码块1
实例代码块2
构造器1
实例代码块1
实例代码块2
构造器2

> 说明：

  • 静态代码块按照在类中出现的顺序执行
  • 实例代码块按照在类中出现的顺序执行
  • 实例代码块在构造器之前执行

5. 代码块的应用场景

5.1 静态代码块的应用场景

  • 初始化静态变量
  • 加载配置文件
  • 初始化数据库连接池
  • 执行一次性的静态资源加载
java
public class ConfigLoader {
    private static String configPath;
    static {
        configPath = System.getProperty("config.path");
        if (configPath == null) {
            configPath = "default.properties";
        }
        loadConfig(configPath);
    }
    private static void loadConfig(String path) {
        System.out.println("加载配置文件：" + path);
    }
}

5.2 实例代码块的应用场景

  • 统一初始化多个构造器共用的逻辑
  • 初始化复杂的实例变量
  • 执行每次创建对象时都需要执行的逻辑
java
public class User {
    private String name;
    private int age;
    private String email;
    {
        name = "匿名用户";
        age = 0;
        email = "no-email@example.com";
    }
    public User() {}
    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public User(String name, int age, String email) {
        this.name = name;
        this.age = age;
        this.email = email;
    }
}

类型执行时机执行次数是否依赖对象用途
静态代码块类加载时一次否初始化静态资源
实例代码块创建对象时每次是统一初始化实例变量

> 代码块 = 初始化逻辑的补充机制

> 💡 建议： > - 尽量避免在代码块中写复杂的逻辑 > - 使用代码块时要确保逻辑清晰、易于维护 > - 在实际项目中，可以结合 Builder 模式或 Spring 注入机制来替代部分代码块功能

3. 72

内部类-成员内部类-静态内部类-局部内部类

在 Java 中，内部类（Inner Class） 是指定义在另一个类内部的类。内部类可以访问外部类的所有成员（包括私有成员），并且可以用于实现复杂的封装和设计模式。根据其定义位置和特性，内部类主要分为以下三种类型：

  • 成员内部类（Member Inner Class）
  • 静态内部类（Static Inner Class）
  • 局部内部类（Local Inner Class）

1. 成员内部类（Member Inner Class）

1.1 定义方式

java
public class OuterClass {
    private String outerField = "外部类字段";
    public class InnerClass {
        private String innerField = "内部类字段";
        public void display() {
            System.out.println("外部类字段：" + outerField);
            System.out.println("内部类字段：" + innerField);
        }
    }
}

1.2 特点

  • 成员内部类是外部类的一个成员，可以访问外部类的所有成员（包括私有成员）
  • 成员内部类不能定义静态成员（除了 static final 常量）
  • 成员内部类的对象必须依赖于外部类的对象才能创建

1.3 创建方式

java
public class Test {
    public static void main(String[] args) {
        // 创建外部类对象
        OuterClass outer = new OuterClass();
        // 创建成员内部类对象
        OuterClass.InnerClass inner = outer.new InnerClass();
        // 调用方法
        inner.display();
    }
}

1.4 示例

java
public class OuterClass {
    private String name = "张三";
    public class InnerClass {
        private String innerName = "李四";
        public void show() {
            System.out.println("外部类姓名：" + name);
            System.out.println("内部类姓名：" + innerName);
        }
    }
    public InnerClass getInner() {
        return new InnerClass();
    }
}
public class Test {
    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        OuterClass.InnerClass inner = outer.getInner();
        inner.show();
    }
}

> 说明：

  • 成员内部类可以访问外部类的私有成员
  • 成员内部类的对象必须通过外部类的对象创建

2. 静态内部类（Static Inner Class）

2.1 定义方式

java
public class OuterClass {
    private String outerField = "外部类字段";
    public static class StaticInnerClass {
        private String innerField = "内部类字段";
        public void display() {
            System.out.println("内部类字段：" + innerField);
            // 注意：不能直接访问外部类的非静态成员
            // System.out.println(outerField); // 编译错误
        }
    }
}

2.2 特点

  • 使用 static 关键字修饰
  • 静态内部类可以访问外部类的静态成员
  • 静态内部类的对象可以直接创建，不需要外部类的对象
  • 静态内部类可以定义静态成员

2.3 创建方式

java
public class Test {
    public static void main(String[] args) {
        // 创建静态内部类对象
        OuterClass.StaticInnerClass inner = new OuterClass.StaticInnerClass();
        inner.display();
    }
}

2.4 示例

java
public class OuterClass {
    private static String staticField = "静态字段";
    private String instanceField = "实例字段";
    public static class StaticInnerClass {
        private String innerField = "内部类字段";
        public void display() {
            System.out.println("静态字段：" + staticField);
            System.out.println("内部类字段：" + innerField);
            // System.out.println(instanceField); // 编译错误
        }
    }
}
public class Test {
    public static void main(String[] args) {
        OuterClass.StaticInnerClass inner = new OuterClass.StaticInnerClass();
        inner.display();
    }
}

> 说明：

  • 静态内部类不能访问外部类的非静态成员
  • 静态内部类的对象可以直接创建

3. 局部内部类（Local Inner Class）

3.1 定义方式

java
public class OuterClass {
    private String outerField = "外部类字段";
    public void method() {
        final String localVar = "局部变量";
        class LocalInnerClass {
            private String innerField = "内部类字段";
            public void display() {
                System.out.println("外部类字段：" + outerField);
                System.out.println("局部变量：" + localVar);
            }
        }
        LocalInnerClass inner = new LocalInnerClass();
        inner.display();
    }
}

3.2 特点

  • 定义在方法内部
  • 可以访问外部类的成员和方法参数
  • 可以访问方法中的 final 或 effectively final 局部变量
  • 不能定义访问修饰符（如 public、private）
  • 不能定义静态成员

3.3 示例

java
public class OuterClass {
    private String name = "张三";
    public void createPrinter(String message) {
        final String prefix = "打印内容：";
        class Printer {
            public void print() {
                System.out.println(prefix + message);
                System.out.println("外部类姓名：" + name);
            }
        }
        Printer printer = new Printer();
        printer.print();
    }
    public static void main(String[] args) {
        OuterClass outer = new OuterClass();
        outer.createPrinter("Hello World");
    }
}

> 说明：

  • 局部内部类可以访问外部类的成员
  • 局部内部类可以访问方法中的 final 或 effectively final 局部变量
  • 局部内部类的作用域仅限于定义它的方法

4. 内部类的应用场景

4.1 成员内部类的应用场景

  • 实现复杂的封装和数据隐藏
  • 实现回调机制
  • 实现事件监听器
java
public class Button {
    private String label;
    public Button(String label) {
        this.label = label;
    }
    public class ClickListener {
        public void onClick() {
            System.out.println("按钮 " + label + " 被点击");
        }
    }
    public ClickListener getClickListener() {
        return new ClickListener();
    }
}

4.2 静态内部类的应用场景

  • 实现工具类或配置类
  • 实现 Builder 模式
  • 实现单例模式
java
public class Singleton {
    private static class SingletonHolder {
        private static final Singleton INSTANCE = new Singleton();
    }
    private Singleton() {}
    public static Singleton getInstance() {
        return SingletonHolder.INSTANCE;
    }
}

4.3 局部内部类的应用场景

  • 实现临时的回调逻辑
  • 实现方法内部的复杂逻辑封装
  • 实现事件监听器
java
public class EventProcessor {
    public void processEvent(String event) {
        class EventHandler {
            public void handle() {
                System.out.println("处理事件：" + event);
            }
        }
        EventHandler handler = new EventHandler();
        handler.handle();
    }
}

5. 总结

类型是否依赖外部类对象是否可以定义静态成员是否可以访问外部类成员用途
成员内部类是否（除了 static final）是封装、回调、事件监听
静态内部类否是只能访问静态成员工具类、配置类、单例模式
局部内部类否否是（外部类成员和 final 局部变量）方法内部逻辑封装、临时回调

> 内部类 = 封装 + 复用 + 灵活性

> 💡 建议： > - 在实际项目中，优先使用静态内部类来实现工具类或配置类 > - 在需要访问外部类成员时，使用成员内部类 > - 在方法内部需要临时逻辑时，使用局部内部类 > - 避免过度使用内部类，保持代码简洁清晰

匿名内部类-认识-使用形式

在 Java 中，匿名内部类（Anonymous Inner Class） 是一种特殊的内部类，它没有名字，通常用于简化代码，特别是在实现接口或继承抽象类时。匿名内部类常用于事件监听、回调函数等场景，能够快速创建一个类的实例并立即使用。

1. 什么是匿名内部类？

匿名内部类是定义在方法内部或类中的一种内部类，它没有名字，且只能创建一个实例。它通常用于实现接口或继承抽象类，从而快速创建一个类的实例。

1.1 特点

  • 没有名字
  • 只能创建一个实例
  • 通常用于实现接口或继承抽象类
  • 可以访问外部类的成员和方法参数
  • 不能定义访问修饰符（如 public、private）
  • 不能定义静态成员

函数式编程-方法引用

在 Java 8 中引入了函数式编程的支持，其中 方法引用（Method Reference） 是一种简洁的语法，用于引用已经存在的方法，而无需显式地编写 Lambda 表达式。方法引用可以看作是 Lambda 表达式的语法糖，使代码更加简洁、可读性更高。

1. 什么是方法引用？

方法引用是一种引用已有方法的语法，它允许开发者将方法作为参数传递，而无需重新实现该方法。方法引用的语法格式如下：

java
类名::方法名

方法引用可以引用以下几种方法：

  • 静态方法
  • 实例方法
  • 构造器
  • 特定对象的实例方法

2. 方法引用的类型

2.1 引用静态方法

语法：类名::方法名

java
public class MathUtils {
    public static int add(int a, int b) {
        return a + b;
    }
    public static void main(String[] args) {
        // 使用方法引用
        BinaryOperator<Integer> add = MathUtils::add;
        // 调用方法
        System.out.println(add.apply(3, 5)); // 输出 8
    }
}

> 说明：

  • MathUtils::add 引用了 MathUtils 类中的静态方法 add
  • 方法引用可以替代 Lambda 表达式 Integer::add

2.2 引用实例方法

语法：对象名::方法名

java
public class StringProcessor {
    public String toUpperCase(String str) {
        return str.toUpperCase();
    }
    public static void main(String[] args) {
        StringProcessor processor = new StringProcessor();
        // 使用方法引用
        Function<String, String> toUpperCase = processor::toUpperCase;
        // 调用方法
        System.out.println(toUpperCase.apply("hello")); // 输出 HELLO
    }
}

> 说明：

  • processor::toUpperCase 引用了 processor 对象的 toUpperCase 方法
  • 方法引用可以替代 Lambda 表达式 (str) -> processor.toUpperCase(str)

2.3 引用特定对象的实例方法

语法：对象名::方法名

java

  • 方法引用可以替代 Lambda 表达式 (str) -> processor.toUpperCase(str)

2.4 引用构造器

语法：类名::new

java
public class Person {
    private String name;
    private int age;
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    public Person(String name) {
        this.name = name;
        this.age = 0;
    }
    public Person() {
        this.name = "Unknown";
        this.age = 0;
    }
    @Override
    public String toString() {
        return "Person{name='" + name + "', age=" + age + "}";
    }
    public static void main(String[] args) {
        // 使用方法引用创建对象
        Supplier<Person> supplier = Person::new;
        // 调用方法
        System.out.println(supplier.get()); // 输出 Person{name='Unknown', age=0}
        // 使用方法引用创建对象
        Function<String, Person> function = Person::new;
        // 调用方法
        System.out.println(function.apply("Alice")); // 输出 Person{name='Alice', age=0}
    }
}

> 说明：

  • Person::new 引用了 Person 类的无参构造器
  • 方法引用可以替代 Lambda 表达式 () -> new Person()

import java.util.*;
public class HuarongDao {
    private int[][] board;
    private int emptyRow;
    private int emptyCol;
    public HuarongDao() {
        this.board = new int[4][4];
        this.emptyRow = 3;
        this.emptyCol = 3;
        generateBoard();
    }
    /**
     * 生成初始棋盘
     */
    public void generateBoard() {
        List<Integer> numbers = new ArrayList<>();
        for (int i = 1; i <= 15; i++) {
            numbers.add(i);
        }
        numbers.add(0); // 空格
        Collections.shuffle(numbers);
        int index = 0;
        for (int i = 0; i < 4; i++) {
            for (int j = 0; j < 4; j++) {
                board[i][j] = numbers.get(index++);
            }
        }
        // 找到空格的位置
        for (int i = 0; i < 4; i++) {
            for (int j = 0; j < 4; j++) {
                if (board[i][j] == 0) {
                    emptyRow = i;
                    emptyCol = j;
                    break;
                }
            }
        }
    }
    /**
     * 显示当前棋盘
     */
    public void displayBoard() {
        System.out.println("=== 数字华容道 ===");
        for (int i = 0; i < 4; i++) {
            for (int j = 0; j < 4; j++) {
                if (board[i][j] == 0) {
                    System.out.print("   ");
                } else {
                    System.out.printf("%2d ", board[i][j]);
                }
            }
            System.out.println();
        }
    }
    /**
     * 移动数字
     * @param direction 1: 上, 2: 下, 3: 左, 4: 右
     */
    public void move(int direction) {
        int newRow = emptyRow;
        int newCol = emptyCol;
        switch (direction) {
            case 1: // 上
                newRow--;
                break;
            case 2: // 下
                newRow++;
                break;
            case 3: // 左
                newCol--;
                break;
            case 4: // 右
                newCol++;
                break;
            default:
                System.out.println("无效方向！");
                return;
        }
        // 检查是否越界
        if (newRow < 0 
newRow >= 4newCol < 0
 newCol >= 4) {
            System.out.println("移动越界！");
            return;
        }
        // 交换空格和目标数字
        board[emptyRow][emptyCol] = board[newRow][newCol];
        board[newRow][newCol] = 0;
        // 更新空格位置
        emptyRow = newRow;
        emptyCol = newCol;
    }
    /**
     * 判断是否胜利
     * @return 是否胜利
     */
    public boolean isWin() {
        int expected = 1;
        for (int i = 0; i < 4; i++) {
            for (int j = 0; j < 4; j++) {
                if (i == 3 && j == 3) {
                    return board[i][j] == 0;
                }
                if (board[i][j] != expected++) {
                    return false;
                }
            }
        }
        return true;
    }
    /**
     * 主函数
     */
    public static void main(String[] args) {
        HuarongDao game = new HuarongDao();
        Scanner scanner = new Scanner(System.in);
        System.out.println("欢迎来到数字华容道！");
        System.out.println("请输入移动方向：1-上, 2-下, 3-左, 4-右");
        System.out.println("输入 0 退出游戏");
        while (true) {
            game.displayBoard();
            if (game.isWin()) {
                System.out.println("恭喜你，游戏胜利！");
                break;
            }
            System.out.print("请输入方向：");
            int direction = scanner.nextInt();
            if (direction == 0) {
                System.out.println("感谢游戏！");
                break;
            }
            game.move(direction);
        }
        scanner.close();
    }
}
