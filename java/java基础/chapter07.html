final 关键字 - 详解 Java 常量

final 关键字 - 详解 Java 常量

在 Java 中，final 是一个关键字，用于定义常量或不可变的变量。它确保变量的值在初始化后不能被修改，从而提高程序的安全性和可读性。

1. final 关键字的基本语法

1.1 定义常量

java
final 数据类型 变量名 = 值;
  • final 修饰的变量必须在声明时初始化
  • 初始化后，值不能被修改
java
final int MAX_SIZE = 100;
final double PI = 3.14159;
final String NAME = "张三";

> 说明：

  • MAX_SIZE 是一个常量，值为 100
  • PI 是一个常量，值为 3.14159
  • NAME 是一个常量，值为 "张三"

2. final 的应用场景

2.1 定义常量

  • 用于定义程序中不会改变的值
  • 提高代码的可读性和安全性
java
public class Constants {
    public static final int MAX_USERS = 1000;
    public static final double TAX_RATE = 0.05;
    public static final String DEFAULT_NAME = "匿名用户";
}

> 说明：

  • MAX_USERS：最大用户数
  • TAX_RATE：税率
  • DEFAULT_NAME：默认用户名

2.2 定义方法参数

  • final 修饰的方法参数表示参数在方法体内不能被修改
java
public void printName(final String name) {
    // name = "李四"; // ❌ 编译错误：不能修改 final 参数
    System.out.println(name);
}

> 说明：

  • name 是 final 参数，不能在方法体内修改
  • 提高方法的安全性

2.3 定义局部变量

  • final 修饰的局部变量表示变量在初始化后不能被修改
java
public void calculate() {
    final int x = 10;
    // x = 20; // ❌ 编译错误：不能修改 final 局部变量
    System.out.println(x);
}

> 说明：

  • x 是 final 局部变量，值为 10
  • 不能在方法体内修改 x 的值

3. final 的注意事项

3.1 final 变量必须初始化

  • final 修饰的变量必须在声明时初始化
  • 否则会编译错误
java
final int x; // ❌ 编译错误：必须初始化

> 说明：

  • final 变量必须在声明时初始化
  • 否则会编译错误

3.2 final 变量的值不能被修改

  • final 修饰的变量值不能被修改
  • 否则会编译错误
java
final int x = 10;
x = 20; // ❌ 编译错误：不能修改 final 变量

> 说明：

  • x 是 final 变量，值为 10
  • 不能在方法体内修改 x 的值

3.3 final 修饰的引用类型

  • final 修饰的引用类型表示引用不能被修改
  • 但引用指向的对象可以被修改
java
final List<String> list = new ArrayList<>();
list.add("张三"); // ✅ 可以修改 list 的内容
list = new ArrayList<>(); // ❌ 编译错误：不能修改 final 引用

> 说明：

  • list 是 final 引用，不能指向其他对象
  • 但 list 指向的对象可以被修改

4. final 的最佳实践

4.1 使用 final 定义常量

  • 优先使用 final 定义常量
  • 提高代码的可读性和安全性
java
public class Constants {
    public static final int MAX_USERS = 1000;
    public static final double TAX_RATE = 0.05;

> 说明：

  • MAX_USERS：最大用户数
  • TAX_RATE：税率
  • DEFAULT_NAME：默认用户名

4.2 使用 final 修饰方法参数

  • 优先使用 final 修饰方法参数
  • 提高方法的安全性
java
public void printName(final String name) {
    System.out.println(name);
}

> 说明：

  • name 是 final 参数，不能在方法体内修改
  • 提高方法的安全性

4.3 使用 final 修饰局部变量

  • 优先使用 final 修饰局部变量
  • 提高代码的可读性和安全性
java
public void calculate() {
    final int x = 10;
    System.out.println(x);
}

> 说明：

  • x 是 final 局部变量，值为 10
  • 不能在方法体内修改 x 的值

值不能被修改final 修饰的变量值不能被修改

单例设计模式 - 懒汉式单例与饿汉式单例

单例设计模式 - 懒汉式单例与饿汉式单例

在 Java 中，单例设计模式（Singleton Pattern） 是一种创建型设计模式，它确保一个类只有一个实例，并提供一个全局访问点。单例模式常用于管理共享资源，如数据库连接池、日志对象、配置管理等。

1. 单例模式的基本要求

  • 构造函数私有化：防止外部直接创建实例
  • 提供全局访问点：通过静态方法获取实例
  • 线程安全：在多线程环境下，确保只有一个实例

2. 饿汉式单例（Eager Initialization）

2.1 定义

饿汉式单例在类加载时就创建实例，因此它是线程安全的，但可能会浪费内存。

java
public class EagerSingleton {
    // 饿汉式：在类加载时创建实例
    private static final EagerSingleton instance = new EagerSingleton();
    // 私有构造函数
    private EagerSingleton() {}
    // 提供全局访问点
    public static EagerSingleton getInstance() {
        return instance;
    }
}

2.2 特点

  • 线程安全：在类加载时创建实例，无需同步
  • 内存浪费：即使不使用，也会创建实例
  • 立即加载：类加载时就创建实例

3. 懒汉式单例（Lazy Initialization）

3.1 定义

懒汉式单例在第一次调用 getInstance() 方法时才创建实例，因此它节省了内存，但需要考虑线程安全问题。

java
public class LazySingleton {
    // 懒汉式：在第一次调用时创建实例
    private static LazySingleton instance;
    // 私有构造函数
    private LazySingleton() {}
    // 提供全局访问点
    public static LazySingleton getInstance() {
        if (instance == null) {
            instance = new LazySingleton();
        }
        return instance;
    }
}

3.2 特点

  • 内存节省：只有在需要时才创建实例
  • 线程安全问题：在多线程环境下，可能创建多个实例
  • 延迟加载：只有在需要时才创建实例

4. 懒汉式单例的线程安全问题

4.1 问题描述

在多线程环境下，多个线程可能同时进入 getInstance() 方法，导致创建多个实例。

java
public class LazySingleton {
    private static LazySingleton instance;
    private LazySingleton() {}
    public static LazySingleton getInstance() {
        if (instance == null) {
            instance = new LazySingleton(); // 可能创建多个实例
        }
        return instance;
    }
}

> 说明：

  • 多个线程可能同时进入 if (instance == null) 判断
  • 导致创建多个实例

4.2 解决方案

#### 4.2.1 使用 synchronized 关键字

java
public class LazySingleton {
    private static LazySingleton instance;
    private LazySingleton() {}
    public static synchronized LazySingleton getInstance() {
        if (instance == null) {
            instance = new LazySingleton();
        }
        return instance;
    }
}

> 说明：

  • synchronized 关键字确保线程安全
  • 但性能较差，因为每次调用都会同步
#### 4.2.2 使用双重检查锁定（Double-Checked Locking）

java
public class LazySingleton {
    private static volatile LazySingleton instance;
    private LazySingleton() {}
    public static LazySingleton getInstance() {
        if (instance == null) {
            synchronized (LazySingleton.class) {
                if (instance == null) {
                    instance = new LazySingleton();
                }
            }
        }
        return instance;
    }
}

> 说明：

  • volatile 关键字确保可见性和有序性
  • 双重检查锁定提高性能

5. 饿汉式 vs 懒汉式

特性饿汉式懒汉式
实例创建时机类加载时第一次调用时
内存使用可能浪费内存节省内存
线程安全线程安全需要同步
性能高低（同步开销）
适用场景实例创建成本高实例创建成本低

6. 总结

特点说明
单例模式确保一个类只有一个实例
饿汉式类加载时创建实例，线程安全
懒汉式第一次调用时创建实例，需要同步
线程安全饿汉式天然线程安全，懒汉式需要同步

> 单例模式 = 全局唯一实例的保证

> 💡 建议： > - 在实际项目中，优先使用饿汉式单例 > - 在复杂逻辑中，考虑使用 Builder 模式 > - 在测试中，使用 Mockito 模拟单例行为

枚举 - 认识枚举类

枚举 - 认识枚举类

在 Java 中，枚举（Enum） 是一种特殊的数据类型，用于定义一组固定的常量。枚举类是 Java 5 引入的一种特殊类，它继承自 java.lang.Enum，并提供了类型安全的常量定义方式。

1. 枚举的基本语法

1.1 定义枚举类

java
public enum Color {
    RED, GREEN, BLUE;
}

> 说明：

  • enum 关键字用于定义枚举类
  • 枚举常量用逗号分隔
  • 枚举类默认继承自 java.lang.Enum

1.2 使用枚举类

java
public class Main {
    public static void main(String[] args) {
        Color color = Color.RED;
        System.out.println(color); // 输出: RED
    }
}

> 说明：

  • Color.RED 是枚举常量
  • 枚举常量是 Color 类型的实例

2. 枚举的特点

2.1 类型安全

枚举类提供了类型安全的常量定义方式，避免了使用字符串或整数定义常量的错误。

java
public enum Status {
    ACTIVE, INACTIVE, PENDING;
}
public class Main {
    public static void main(String[] args) {
        Status status = Status.ACTIVE;
        System.out.println(status); // 输出: ACTIVE
    }
}

> 说明：

  • Status.ACTIVE 是 Status 类型的实例
  • 避免了使用字符串或整数定义常量的错误

2.2 唯一性

枚举常量是唯一的，不能重复。

java
public enum Color {
    RED, GREEN, BLUE;
}

> 说明：

  • Color.RED 是唯一的
  • 不能重复定义 RED

2.3 可以添加方法和字段

枚举类可以像普通类一样添加方法和字段。

java
public enum Color {
    RED("红色"), GREEN("绿色"), BLUE("蓝色");
    private String description;
    Color(String description) {
        this.description = description;
    }
    public String getDescription() {
        return description;
    }
}

> 说明：

  • 枚举类可以有构造函数
  • 枚举类可以有方法和字段

3. 枚举的应用场景

3.1 定义常量

  • 用于定义一组固定的常量
  • 提高代码的可读性和安全性
java
public enum Weekday {
    MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY;
}

> 说明：

  • Weekday.MONDAY 是 Weekday 类型的实例
  • 用于表示一周中的某一天

3.2 状态管理

  • 用于管理对象的状态
  • 提高代码的可读性和安全性
java
public enum OrderStatus {
    PENDING, SHIPPED, DELIVERED, CANCELLED;
}

> 说明：

  • OrderStatus.PENDING 是 OrderStatus 类型的实例
  • 用于表示订单的状态

3.3 配置管理

  • 用于管理配置项
  • 提高代码的可读性和安全性
java
public enum DatabaseType {
    MYSQL, POSTGRESQL, ORACLE;
}

> 说明：

  • DatabaseType.MYSQL 是 DatabaseType 类型的实例
  • 用于表示数据库类型

4. 枚举的注意事项

4.1 枚举常量是唯一的

  • 枚举常量是唯一的，不能重复
  • 枚举常量是 Enum 类型的实例
java
public enum Color {
    RED, GREEN, BLUE;
}

> 说明：

  • Color.RED 是唯一的
  • 不能重复定义 RED

4.2 枚举类不能继承

  • 枚举类不能继承其他类
  • 枚举类默认继承自 java.lang.Enum
java
public enum Color {
    RED, GREEN, BLUE;
}

> 说明：

  • 枚举类不能继承其他类
  • 枚举类默认继承自 java.lang.Enum

4.3 枚举类不能被实例化

  • 枚举类不能通过 new 关键字实例化
  • 枚举常量是 Enum 类型的实例
java
public enum Color {
    RED, GREEN, BLUE;
}

> 说明：

  • Color.RED 是 Color 类型的实例
  • 不能通过 new Color() 实例化

5. 总结

特点说明
枚举定义一组固定的常量
类型安全提供类型安全的常量定义方式
唯一性枚举常量是唯一的
可扩展性枚举类可以添加方法和字段

> 枚举 = 类型安全的常量定义
